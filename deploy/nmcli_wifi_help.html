🔍 Основные команды просмотра

Проверка статуса NetworkManager

sudo systemctl status NetworkManager
Показывает состояние службы NetworkManager - запущена ли, активна ли, есть ли ошибки

Просмотр всех устройств

nmcli device status
Отображает список всех сетевых устройств и их состояние
Пример вывода: 
DEVICE  TYPE      STATE         CONNECTION
wlan0   wifi      connected     MyHomeWiFi
eth0    ethernet  unavailable   --
lo      loopback  unmanaged     --

Список доступных WiFi сетей

nmcli device wifi list
Показывает все доступные WiFi сети в радиусе действия с информацией о сигнале, безопасности и частоте
Пример вывода: 
IN-USE  BSSID              SSID           MODE   CHAN  RATE       SIGNAL  BARS  SECURITY
*       00:11:22:33:44:55  MyHomeWiFi     Infra  6     54 Mbit/s  85      ▂▄▆█  WPA2
        AA:BB:CC:DD:EE:FF  NeighborWiFi   Infra  11    54 Mbit/s  45      ▂▄__  WPA2
        FF:EE:DD:CC:BB:AA  CafeNetwork    Infra  1     54 Mbit/s  30      ▂___  --
Расшифровка столбцов:
  • IN-USE - * означает текущее подключение
  • BSSID - MAC-адрес точки доступа
  • SSID - имя сети
  • MODE - режим (Infra = Infrastructure)
  • CHAN - канал WiFi (1-14)
  • RATE - максимальная скорость
  • SIGNAL - уровень сигнала (0-100)
  • BARS - визуальный индикатор сигнала
  • SECURITY - тип шифрования (WPA2, WPA3, -- = открытая)

Обновить список сетей

nmcli device wifi rescan
Повторное сканирование доступных WiFi сетей (полезно, если не видите нужную сеть)

Показать текущее подключение

nmcli connection show --active
Отображает только активные соединения прямо сейчас

Быстрая проверка статуса WiFi

nmcli -t -f DEVICE,STATE device
Компактный вывод устройств и их состояния
💡 Уровни сигнала WiFi:
  • 80-100% - Отличный сигнал ▂▄▆█
  • 60-79% - Хороший сигнал ▂▄▆_
  • 40-59% - Средний сигнал ▂▄__
  • 20-39% - Слабый сигнал ▂___
  • 0-19% - Очень слабый сигнал ____

🔗 Подключение к WiFi сети

Подключение к открытой сети

sudo nmcli device wifi connect "SSID_СЕТИ"
Подключается к открытой WiFi сети (без пароля)

Подключение к защищённой сети

sudo nmcli device wifi connect "SSID_СЕТИ" password "ВАШ_ПАРОЛЬ"
Подключается к WiFi сети с паролем (WPA/WPA2/WPA3)

Подключение к скрытой сети

sudo nmcli device wifi connect "SSID_СЕТИ" password "ВАШ_ПАРОЛЬ" hidden yes
Подключается к скрытой WiFi сети (которая не транслирует свой SSID)

Подключение с указанием интерфейса

sudo nmcli device wifi connect "SSID_СЕТИ" password "пароль" ifname wlan0
Подключается через конкретный WiFi адаптер (полезно при наличии нескольких адаптеров)

Подключение по BSSID

sudo nmcli device wifi connect "SSID_СЕТИ" password "пароль" bssid 00:11:22:33:44:55
Подключается к конкретной точке доступа по MAC-адресу (полезно когда несколько AP с одинаковым SSID)
📖 Пример: Подключение к домашней сети 
# Сканируем доступные сети
nmcli device wifi list

# Находим нашу сеть "MyHomeWiFi"
# Подключаемся с паролем
sudo nmcli device wifi connect "MyHomeWiFi" password "mySecretPassword123"

# Проверяем подключение
nmcli connection show --active

# Вывод:
# NAME         UUID                                  TYPE  DEVICE
# MyHomeWiFi   12345678-1234-1234-1234-123456789012  wifi  wlan0
📖 Пример: Подключение к скрытой сети офиса 
# Сеть "OfficeHidden" не отображается в списке
nmcli device wifi list
# Её там нет!

# Подключаемся напрямую, указав hidden yes
sudo nmcli device wifi connect "OfficeHidden" password "OfficePass2024" hidden yes

# Успех! Проверяем
ping -c 4 8.8.8.8
📖 Пример: Подключение к публичной сети без пароля 
# В кафе видим открытую сеть "CafeFreeWiFi"
nmcli device wifi list | grep -i cafe

# Подключаемся без пароля
sudo nmcli device wifi connect "CafeFreeWiFi"

# Открываем браузер для авторизации через портал
⚠️ Безопасность:
  • Пароль передаётся в командной строке и может сохраниться в истории команд
  • Используйте history -c после ввода пароля или пробел перед командой
  • Не подключайтесь к неизвестным открытым сетям без VPN
  • Предпочитайте сети с WPA3, затем WPA2
💡 Типы безопасности WiFi:
  • WPA3 - современный, самый безопасный стандарт
  • WPA2 - распространённый, достаточно безопасный
  • WPA - устаревший, слабая защита
  • WEP - очень старый, легко взламывается, не используйте!
  • Открытая - без шифрования, любой видит ваш трафик

🛠️ Устранение неполадок

Перезапуск NetworkManager

sudo systemctl restart NetworkManager
Перезапускает службу NetworkManager (часто помогает при проблемах с подключением)

Отключение устройства

nmcli device disconnect wlan0
Отключает WiFi устройство

Повторное подключение устройства

nmcli device connect wlan0
Подключает WiFi устройство обратно

Проверка логов

journalctl -u NetworkManager -f
Просмотр логов NetworkManager в реальном времени (для диагностики проблем)

Просмотр последних 100 строк логов

journalctl -u NetworkManager -n 100
Показывает последние 100 записей журнала

Логи за последний час

journalctl -u NetworkManager --since "1 hour ago"
Показывает логи NetworkManager за последний час

Проверка драйвера WiFi адаптера

lspci -k | grep -A 3 Network
Показывает информацию о WiFi адаптере и используемом драйвере

Проверка состояния интерфейса

ip link show wlan0
Показывает состояние сетевого интерфейса

Проверка IP адреса

ip addr show wlan0
Показывает IP адрес и другую информацию об интерфейсе

Тест подключения

ping -c 4 8.8.8.8
Проверяет подключение к интернету (пингует Google DNS)

Проверка DNS

nslookup google.com
Проверяет работу DNS резолвера

Трассировка маршрута

traceroute google.com
Показывает путь пакетов до назначения
📖 Сценарий 1: WiFi не подключается 
# Проблема: не можете подключиться к сети

# Шаг 1: Проверяем, включен ли WiFi
nmcli radio wifi
# Если "disabled", включаем:
nmcli radio wifi on

# Шаг 2: Проверяем статус адаптера
nmcli device status
# Должно быть: wlan0  wifi  disconnected  --

# Шаг 3: Сканируем сети
nmcli device wifi list
# Находим нашу сеть в списке

# Шаг 4: Пробуем подключиться
sudo nmcli device wifi connect "MyWiFi" password "password123"

# Шаг 5: Если не работает, смотрим логи
journalctl -u NetworkManager -n 50 --no-pager

# Шаг 6: Перезапускаем NetworkManager
sudo systemctl restart NetworkManager

# Шаг 7: Пробуем снова
sudo nmcli device wifi connect "MyWiFi" password "password123"
📖 Сценарий 2: Подключено, но нет интернета 
# Проблема: WiFi подключен, но интернет не работает

# Шаг 1: Проверяем подключение
nmcli connection show --active
# WiFi должен быть в списке

# Шаг 2: Проверяем IP адрес
ip addr show wlan0
# Должен быть IP типа 192.168.x.x или 10.x.x.x

# Шаг 3: Пингуем роутер
ping -c 4 192.168.1.1
# Если работает - проблема не в WiFi

# Шаг 4: Пингуем внешний IP
ping -c 4 8.8.8.8
# Если работает - проблема в DNS

# Шаг 5: Проверяем DNS
nslookup google.com
# Если не работает - меняем DNS

# Шаг 6: Устанавливаем Google DNS
sudo nmcli connection modify "MyWiFi" ipv4.dns "8.8.8.8 8.8.4.4"
sudo nmcli connection down "MyWiFi"
sudo nmcli connection up "MyWiFi"

# Шаг 7: Проверяем снова
ping -c 4 google.com
📖 Сценарий 3: Слабый сигнал WiFi 
# Проблема: медленное подключение, обрывы

# Шаг 1: Проверяем уровень сигнала
nmcli device wifi list
# Смотрим на SIGNAL и BARS нашей сети

# Шаг 2: Проверяем помехи
nmcli device wifi list | grep "CHAN"
# Смотрим, какие каналы заняты

# Если много сетей на одном канале (например, 6):
# - Роутер на канале 6: 5 сетей
# - Роутер на канале 11: 2 сети
# Рекомендуется переключить роутер на менее загруженный канал

# Шаг 3: Проверяем скорость
sudo apt install speedtest-cli
speedtest-cli

# Шаг 4: Проверяем качество соединения
ping -c 100 192.168.1.1
# Смотрим на % потерь пакетов (packet loss)

# Если потери > 5% - проблема с сигналом
# Решение: переместите роутер, используйте репитер или поменяйте канал
📖 Сценарий 4: Автоматическое переподключение не работает 
# Проблема: после перезагрузки WiFi не подключается автоматически

# Шаг 1: Проверяем настройку автоподключения
nmcli connection show "MyWiFi" | grep autoconnect
# Должно быть: connection.autoconnect: yes

# Шаг 2: Если "no", включаем автоподключение
nmcli connection modify "MyWiFi" connection.autoconnect yes

# Шаг 3: Устанавливаем приоритет
nmcli connection modify "MyWiFi" connection.autoconnect-priority 10

# Шаг 4: Перезагружаемся для проверки
sudo reboot

# После перезагрузки проверяем:
nmcli connection show --active
📖 Сценарий 5: Ошибка "Secrets were required, but not provided" 
# Проблема: ошибка при подключении о недостающих секретах

# Это означает, что пароль не сохранён или неверен

# Шаг 1: Удаляем старое соединение
nmcli connection delete "MyWiFi"

# Шаг 2: Создаём новое подключение с правильным паролем
sudo nmcli device wifi connect "MyWiFi" password "correctPassword123"

# Шаг 3: Проверяем
nmcli connection show --active
⚠️ Распространённые проблемы:
  • WiFi постоянно отключается - проверьте power management: sudo iwconfig wlan0 power off
  • "Device not ready" - драйвер не загружен, проверьте: sudo dmesg | grep firmware
  • Медленная скорость - помехи или слабый сигнал, смените канал на роутере
  • Не видит 5GHz сети - ваш адаптер может не поддерживать 5GHz
  • "Connection failed" - неверный пароль или несовместимый тип шифрования
💡 Команды диагностики одной строкой:
  • Полная диагностика: nmcli && ip a && ping -c 3 8.8.8.8
  • Сброс WiFi: nmcli radio wifi off && sleep 5 && nmcli radio wifi on
  • Быстрое переподключение: nmcli con down "MyWiFi" && nmcli con up "MyWiFi"
  • Проверка всего стека: ping -c 1 127.0.0.1 && ping -c 1 192.168.1.1 && ping -c 1 8.8.8.8 && ping -c 1 google.com

📖 Сценарий: Настройка статического IP в WiFi сети 
# Подключаемся к сети
sudo nmcli device wifi connect "MyWiFi" password "password123"

# Настраиваем статический IP
nmcli connection modify "MyWiFi" ipv4.method manual
nmcli connection modify "MyWiFi" ipv4.addresses 192.168.1.100/24
nmcli connection modify "MyWiFi" ipv4.gateway 192.168.1.1
nmcli connection modify "MyWiFi" ipv4.dns "8.8.8.8,8.8.4.4"

# Применяем изменения
nmcli connection down "MyWiFi"
nmcli connection up "MyWiFi"

# Проверяем
ip addr show wlan0
📖 Сценарий: Создание скрипта для быстрого переподключения 
# Создаём скрипт reconnect-wifi.sh
cat <<'EOF' > ~/reconnect-wifi.sh
#!/bin/bash
WIFI_NAME="MyHomeWiFi"
echo "Отключение от $WIFI_NAME..."
nmcli connection down "$WIFI_NAME"
sleep 2
echo "Подключение к $WIFI_NAME..."
nmcli connection up "$WIFI_NAME"
echo "Проверка подключения..."
ping -c 3 8.8.8.8 && echo "✓ Подключение успешно!" || echo "✗ Нет подключения"
EOF

# Делаем исполняемым
chmod +x ~/reconnect-wifi.sh

# Используем
~/reconnect-wifi.sh
📖 Сценарий: Подключение к сети с MAC-фильтрацией 
# Узнаём MAC адрес нашего адаптера
ip link show wlan0 | grep "link/ether"

# Допустим, MAC: aa:bb:cc:dd:ee:ff
# Добавляем этот MAC в настройки роутера

# После этого подключаемся обычным способом
sudo nmcli device wifi connect "SecureNetwork" password "password"

# Если нужно изменить MAC адрес (спуфинг):
sudo ip link set wlan0 down
sudo ip link set wlan0 address aa:bb:cc:dd:ee:ff

📖 Сценарий: Создание мобильной точки доступа (hotspot) 
# Создаём точку доступа с именем "MyHotspot" и паролем
nmcli device wifi hotspot ifname wlan0 ssid "MyHotspot" password "HotspotPass123"

# Проверяем
nmcli connection show --active

# Останавливаем hotspot
nmcli connection down Hotspot

# Настройка постоянного hotspot
nmcli connection add type wifi ifname wlan0 \
  con-name MyHotspot \
  autoconnect yes \
  ssid MyHotspot

nmcli connection modify MyHotspot \
  wifi-sec.key-mgmt wpa-psk \
  wifi-sec.psk "HotspotPass123" \
  ipv4.method shared \
  ipv6.method ignore

# Запускаем
nmcli connection up MyHotspot
📖 Сценарий: Экспорт и импорт настроек WiFi 
# Экспорт настроек всех подключений
sudo cp -r /etc/NetworkManager/system-connections /tmp/wifi-backup

# Или конкретного подключения
sudo cp "/etc/NetworkManager/system-connections/MyWiFi.nmconnection" ~/wifi-backup.nmconnection

# Импорт на другом компьютере
sudo cp ~/wifi-backup.nmconnection /etc/NetworkManager/system-connections/
sudo chmod 600 /etc/NetworkManager/system-connections/wifi-backup.nmconnection
sudo systemctl restart NetworkManager

# Проверяем
nmcli connection show
📖 Сценарий: Мониторинг качества WiFi соединения 
# Устанавливаем wavemon для мониторинга
sudo apt install wavemon

# Запускаем мониторинг
wavemon
