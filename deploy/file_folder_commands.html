🗂️ Навигация по файловой системе

Показать текущую директорию

pwd
Выводит полный путь к текущей директории

Перейти в директорию

cd /путь/к/папке
Переходит в указанную директорию

Перейти в домашнюю директорию

cd ~
Переходит в домашнюю папку пользователя

Перейти на уровень выше

cd ..
Переходит на один уровень выше в иерархии папок

Вернуться в предыдущую директорию

cd -
Возвращается в предыдущую директорию

📂 Создание папок

Создать новую папку

mkdir имя_папки
Создаёт новую директорию с указанным именем

Создать несколько папок

mkdir папка1 папка2 папка3
Создаёт несколько папок одновременно

Создать вложенные папки

mkdir -p родитель/дочерняя/внук
Создаёт всю цепочку директорий, включая несуществующие родительские

Создать папку с правами доступа

mkdir -m 755 имя_папки
Создаёт папку с указанными правами доступа
📖 Пример: Создание структуры проекта 
# Создаём структуру для веб-проекта одной командой
mkdir -p myproject/{src,public/{css,js,images},tests,docs}

# Проверяем структуру
tree myproject

# Результат:
# myproject/
# ├── src/
# ├── public/
# │   ├── css/
# │   ├── js/
# │   └── images/
# ├── tests/
# └── docs/
📖 Пример: Создание папок по датам 
# Создаём папки для ежедневных бэкапов
mkdir -p backup/$(date +%Y)/$(date +%m)/$(date +%d)

# Или за несколько дней сразу
for i in {1..7}; do
    mkdir -p "backup/day_$i"
done

# Создаём архивные папки по годам
mkdir -p archive/{2020..2025}/{01..12}

🗑️ Удаление файлов и папок

⚠️ ВНИМАНИЕ: Удаление в Linux необратимо! Файлы не попадают в корзину. Будьте осторожны с командой rm.

Удалить файл

rm файл.txt
Удаляет файл безвозвратно

Удалить несколько файлов

rm файл1.txt файл2.txt файл3.txt
Удаляет несколько файлов

Удалить с подтверждением

rm -i файл.txt
Запрашивает подтверждение перед удалением

Удалить пустую папку

rmdir имя_папки
Удаляет только пустую директорию

Удалить папку со всем содержимым

rm -r имя_папки
Рекурсивно удаляет папку и всё её содержимое

Принудительное удаление

rm -rf имя_папки
Удаляет без подтверждения (используйте с осторожностью!)

Удалить все файлы с расширением

rm *.txt
Удаляет все файлы .txt в текущей папке

🔍 Поиск файлов и папок

Найти файл по имени

find /путь -name "имя_файла.txt"
Ищет файл с точным именем

Найти файл без учёта регистра

find /путь -iname "имя_файла.txt"
Ищет файл игнорируя регистр букв

Найти все файлы с расширением

find /путь -name "*.txt"
Находит все файлы .txt

Найти папки

find /путь -type d -name "имя_папки"
Ищет только директории

Найти файлы по размеру

find /путь -size +100M
Находит файлы больше 100 Мб

Найти файлы изменённые за последние 7 дней

find /путь -mtime -7
Находит файлы изменённые за последнюю неделю

Поиск по содержимому файлов

grep -r "текст_для_поиска" /путь
Рекурсивно ищет текст внутри файлов

Быстрый поиск по базе

locate имя_файла
Быстро находит файл в системной базе (требует updatedb)

Обновление базы locate

sudo updatedb
Обновляет базу данных файлов для команды locate
📖 Пример: Поиск всех конфигурационных файлов 
# Найти все файлы .conf в системе
sudo find / -name "*.conf" 2>/dev/null

# Найти конфигурационные файлы в /etc
find /etc -name "*.conf"

# Найти и показать содержимое
find /etc -name "*.conf" -exec cat {} \;
📖 Пример: Поиск и удаление временных файлов 
# Найти все временные файлы старше 7 дней
find /tmp -type f -mtime +7

# Найти и удалить их
find /tmp -type f -mtime +7 -delete

# Найти большие лог-файлы (больше 100MB)
find /var/log -type f -size +100M

# Найти и сжать большие логи
find /var/log -type f -size +100M -exec gzip {} \;
📖 Пример: Поиск дубликатов файлов 
# Установка fdupes
sudo apt install fdupes

# Поиск дубликатов в папке

# Или через find по размеру
find . -type f -exec md5sum {} \; | sort | uniq -w32 -dD
📖 Пример: Поиск файлов по содержимому

# Найти в файлах Python слово "import"

🔐 Права доступа

Просмотр прав доступа

ls -l файл.txt
Показывает права доступа к файлу

Изменить права (числовой формат)

chmod 755 файл.txt
Устанавливает права rwxr-xr-x (755)

Изменить права (символьный формат)

chmod u+x файл.txt
Добавляет право на выполнение владельцу

Рекурсивно изменить права папки

chmod -R 755 папка
Применяет права ко всем файлам внутри папки

Сменить владельца файла

sudo chown пользователь:группа файл.txt
Изменяет владельца и группу файла

Рекурсивно сменить владельца папки

sudo chown -R пользователь:группа папка
Изменяет владельца для всего содержимого папки
💡 Расшифровка прав доступа:
  • r (read) = 4 - чтение
  • w (write) = 2 - запись
  • x (execute) = 1 - выполнение
  • 755 = rwxr-xr-x (владелец: все права, остальные: чтение и выполнение)
  • 644 = rw-r--r-- (владелец: чтение и запись, остальные: только чтение)
  • 777 = rwxrwxrwx (все права для всех - небезопасно!)
📊 Таблица часто используемых прав доступа:
Числовой Символьный Описание Применение
644 rw-r--r-- Владелец может читать и писать, остальные только читать Обычные файлы
755 rwxr-xr-x Владелец - все права, остальные - чтение и выполнение Директории, исполняемые файлы
700 rwx------ Только владелец имеет все права Приватные директории (~/.ssh)
600 rw------- Только владелец может читать и писать Приватные файлы (ключи SSH)
777 rwxrwxrwx Все могут всё (ОПАСНО!) Только для тестирования
666 rw-rw-rw- Все могут читать и писать Общие файлы данных
555 r-xr-xr-x Все могут читать и выполнять, никто не может изменять Защищенные скрипты
444 r--r--r-- Только чтение для всех Конфигурационные файлы
👥 Специальные биты:
Бит Числовое Описание Пример
SUID 4000 Выполняется с правами владельца chmod 4755 file
/usr/bin/passwd
SGID 2000 Выполняется с правами группы chmod 2755 file
Общие папки проектов
Sticky 1000 Только владелец может удалить файл chmod 1777 /tmp
Временная папка /tmp

📦 Архивирование и сжатие

Создать tar-архив

tar -cvf архив.tar папка
Создаёт tar-архив без сжатия

Создать tar.gz архив

tar -czvf архив.tar.gz папка
Создаёт сжатый tar.gz архив

Распаковать tar-архив

tar -xvf архив.tar
Извлекает содержимое tar-архива

Распаковать tar.gz архив

tar -xzvf архив.tar.gz
Распаковывает сжатый tar.gz архив

Просмотр содержимого архива

tar -tvf архив.tar
Показывает содержимое архива без распаковки

Создать zip-архив

zip -r архив.zip папка
Создаёт zip-архив

Распаковать zip-архив

unzip архив.zip
Распаковывает zip-архив
💡 Пояснение флагов tar:
  • c - создать архив (create)
  • x - извлечь архив (extract)
  • v - подробный вывод (verbose)
  • f - имя файла (file)
  • z - сжатие gzip
  • t - просмотр содержимого (list)
📖 Пример: Резервное копирование домашней директории 
# Полный бэкап домашней папки
tar -czvf ~/backup-home-$(date +%Y%m%d).tar.gz ~

# Бэкап с исключением больших папок
tar -czvf ~/backup-selective.tar.gz \
    --exclude='Downloads' \
    --exclude='*.tmp' \
    --exclude='.cache' \
    --exclude='node_modules' \
    ~/Documents ~/Pictures ~/Music

# Инкрементальный бэкап (только изменённые файлы)
tar -czf backup-incremental.tar.gz \
    --listed-incremental=backup.snar \
    ~/Documents

# Создание бэкапа с проверкой целостности
tar -czf backup.tar.gz ~/Documents && \
tar -tzf backup.tar.gz > /dev/null && \
echo "✓ Архив создан успешно" || \
echo "✗ Ошибка создания архива"
📖 Пример: Работа с зашифрованными архивами 
# Создание зашифрованного архива с gpg
tar -czf - ~/Documents | gpg -c > documents-encrypted.tar.gz.gpg

# Распаковка зашифрованного архива
gpg -d documents-encrypted.tar.gz.gpg | tar -xz

# Или используя 7z с паролем
sudo apt install p7zip-full
7z a -p -mhe=on documents-encrypted.7z ~/Documents

# Распаковка
7z x documents-encrypted.7z
📖 Пример: Разделение больших архивов 
# Создать архив и разделить на части по 100MB
tar -czf - /large/directory | split -b 100M - backup.tar.gz.part

# Объединить части обратно и распаковать
cat backup.tar.gz.part* | tar -xz

# Создание многотомного архива с 7z
7z a -v100m backup.7z /large/directory

# Распаковка многотомного архива
7z x backup.7z.001
📖 Пример: Автоматический ротационный бэкап 
#!/bin/bash
# backup-rotation.sh - Создаёт бэкапы и хранит только последние 7 дней

BACKUP_DIR=~/backups
SOURCE_DIR=~/Documents
MAX_BACKUPS=7

# Создаём директорию для бэкапов
mkdir -p "$BACKUP_DIR"

# Создаём новый бэкап
BACKUP_FILE="$BACKUP_DIR/backup-$(date +%Y%m%d-%H%M%S).tar.gz"
tar -czf "$BACKUP_FILE" "$SOURCE_DIR"

# Проверяем успешность
if [ $? -eq 0 ]; then
    echo "✓ Бэкап создан: $BACKUP_FILE"
    
    # Удаляем старые бэкапы, оставляем только последние MAX_BACKUPS
    cd "$BACKUP_DIR"
    ls -t backup-*.tar.gz | tail -n +$((MAX_BACKUPS + 1)) | xargs -r rm
    
    echo "✓ Старые бэкапы очищены"
else
    echo "✗ Ошибка создания бэкапа"
    exit 1
fi

🎯 Расширенные практические сценарии

📖 Сценарий 1: Массовое переименование файлов 
# Переименовать все .txt в .md
for file in *.txt; do
    mv "$file" "${file%.txt}.md"
done

# Добавить префикс ко всем файлам
for file in *.jpg; do
    mv "$file" "photo_$file"
done

# Изменить регистр имён файлов
for file in *; do
    mv "$file" "$(echo $file | tr '[:upper:]' '[:lower:]')"
done

# Использование rename (если установлен)
sudo apt install rename
rename 's/\.txt$/\.md/' *.txt
rename 'y/A-Z/a-z/' *
📖 Сценарий 2: Создание и восстановление бэкапов 
# Создание бэкапа домашней директории
tar -czvf ~/backup-$(date +%Y%m%d).tar.gz ~/Documents

# Создание инкрементального бэкапа
rsync -avz --delete ~/Documents/ /backup/documents/

# Создание бэкапа с исключениями
tar -czvf backup.tar.gz \
    --exclude='*.tmp' \
    --exclude='node_modules' \
    ~/myproject/

# Восстановление из бэкапа
tar -xzvf backup-20250101.tar.gz -C /restore/location/

# Автоматический бэкап (добавить в crontab)
0 2 * * * tar -czvf /backup/daily-$(date +\%Y\%m\%d).tar.gz ~/Documents
📖 Сценарий 3: Работа с большим количеством файлов 
# Найти и удалить все пустые файлы
find . -type f -empty -delete

# Найти и удалить все пустые папки
find . -type d -empty -delete

# Подсчитать количество файлов каждого типа

#!/bin/bash
# organize-downloads.sh

    xargs -I {} mv {} "$MUSIC_DIR/"

    xargs -I {} mv {} "$VIDEO_DIR/"
