5. Исправление конфигурации Docker

Проблема: Docker пытался использовать недоступное китайское зеркало

docker.mirrors.ustc.edu.cn - no such host

Решение: Удалили неправильное зеркало из конфигурации Docker:

# Очистили файл
sudo nano /etc/docker/daemon.json

# Перезапустили Docker
sudo systemctl restart docker
✅ Результат: Docker работает корректно, статус active (running)

6. Исправление сетевых проблем

Проблема: Docker не мог скачать образы из-за проблем с маршрутизацией

dial tcp 54.209.164.210:443: connect: no route to host

Причина: Два маршрута по умолчанию (eth0 и wlan0), приоритет был у недоступного Ethernet

default via 192.168.1.1 dev eth0 onlink
default via 192.168.3.1 dev wlan0 proto dhcp metric 600

Решение: Удалили маршрут через недоступный роутер eth0:

sudo ip route del default via 192.168.1.1 dev eth0
✅ Результат: Интернет работает через WiFi (wlan0), SSH через прямой кабель eth0 сохранен
📌 Настройка:
  • Orange Pi статический IP: 192.168.1.100 (eth0)
  • Компьютер: 192.168.1.101 (прямое подключение)
  • Интернет: WiFi HUAWEI-R1FUY4 (192.168.3.25)
  • Маршрут по умолчанию: wlan0 → 192.168.3.1

7. Исправление Dockerfile для ARM64

7.1. Backend Dockerfile

Проблема: Образ eclipse-temurin:17-jre-alpine не поддерживает ARM64

failed to resolve source metadata for docker.io/library/eclipse-temurin:17-jre-alpine:
no match for platform in manifest: not found

Решение: Заменили на Amazon Corretto (поддерживает ARM64)

# Было:
FROM eclipse-temurin:17-jdk-alpine AS build
FROM eclipse-temurin:17-jre-alpine

# Стало:
FROM amazoncorretto:17-alpine AS build
FROM amazoncorretto:17-alpine

7.2. Frontend Dockerfile

Проблема: react-scripts не устанавливался в Alpine на ARM64

sh: react-scripts: not found

Причина: Alpine Linux (musl libc) имеет проблемы совместимости с некоторыми npm пакетами на ARM64

Решение: Использовали полный Debian образ для build stage

# Было:
FROM node:18-alpine AS build
RUN npm ci --only=production

# Стало:
FROM node:18 AS build
RUN npm install
📌 Важно: Debian образ больше Alpine, но для build stage это не критично - в финальный образ попадает только статика
✅ Результат: Оба Dockerfile успешно собираются на ARM64

8. Запуск сборки Docker образов

Запущена команда сборки с исправленными Dockerfile:

docker compose build --no-cache
⏱️ Время сборки: 30-60 минут на Orange Pi Zero 3

Что происходит:

  • Скачивание образа PostgreSQL (postgres:15-alpine)
  • Скачивание образа Amazon Corretto 17 (amazoncorretto:17-alpine)
  • Скачивание образа Node.js Debian (node:18)
  • Скачивание образа Nginx (nginx:alpine)
  • Компиляция backend (Maven + Spring Boot)
  • Сборка frontend (npm install + React build)
✅ Результат: Сборка завершена успешно
