📋 Основные команды просмотра

Просмотр всех сетевых интерфейсов

ip link show
Показывает все сетевые интерфейсы и их состояние (UP/DOWN)

Просмотр IP адресов

ip addr show
Отображает IP адреса всех интерфейсов

Короткая форма команды

ip a
Краткая форма команды ip addr show

Информация о конкретном интерфейсе

ip addr show eth0
Показывает информацию только об интерфейсе eth0

Список всех интерфейсов (включая неактивные)

ip link
Показывает физические и виртуальные сетевые интерфейсы

Статистика интерфейсов

ip -s link
Показывает статистику передачи/приёма пакетов
📖 Пример вывода ip addr show: 
1: lo:  mtu 65536 qdisc noqueue state UNKNOWN
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
    inet6 ::1/128 scope host

2: eth0:  mtu 1500 qdisc fq_codel state UP
    link/ether 08:00:27:12:34:56 brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global eth0
    inet6 fe80::a00:27ff:fe12:3456/64 scope link
Расшифровка:
  • lo - loopback интерфейс (локальная петля)
  • eth0 - Ethernet интерфейс
  • UP - интерфейс активен
  • DOWN - интерфейс неактивен
  • mtu 1500 - максимальный размер пакета
  • inet 192.168.1.100/24 - IPv4 адрес с маской /24
  • link/ether - MAC адрес

Просмотр с цветным выводом

ip -c addr show
Включает цветной вывод для лучшей читаемости

Только IPv4 адреса

ip -4 addr show
Показывает только IPv4 адреса

Только IPv6 адреса

ip -6 addr show
Показывает только IPv6 адреса
💡 Типы сетевых интерфейсов:
  • lo - Loopback (петля обратной связи, 127.0.0.1)
  • eth0, eth1 - Ethernet интерфейсы (проводные)
  • wlan0, wlan1 - WiFi интерфейсы (беспроводные)
  • ens33, ens18 - Предсказуемые имена Ethernet (новая схема именования)
  • enp0s3 - PCI Ethernet (en = Ethernet, p0 = порт 0, s3 = слот 3)
  • br0 - Bridge интерфейс (мост)
  • tun0, tap0 - Туннельные интерфейсы (VPN)
  • veth - Virtual Ethernet (Docker, LXC)

🔧 Команда ip (современный подход)

Включение интерфейса

sudo ip link set eth0 up
Активирует сетевой интерфейс eth0

Отключение интерфейса

sudo ip link set eth0 down
Деактивирует сетевой интерфейс eth0

Добавление IP адреса

sudo ip addr add 192.168.1.100/24 dev eth0
Назначает IP адрес 192.168.1.100 с маской /24 интерфейсу eth0

Удаление IP адреса

sudo ip addr del 192.168.1.100/24 dev eth0
Удаляет IP адрес с интерфейса

Очистка всех IP адресов

sudo ip addr flush dev eth0
Удаляет все IP адреса с интерфейса eth0

Изменение MTU

sudo ip link set eth0 mtu 1400
Устанавливает максимальный размер пакета (MTU) в 1400 байт

Изменение MAC адреса

sudo ip link set eth0 address 00:11:22:33:44:55
Изменяет MAC адрес интерфейса (MAC spoofing)

Добавление нескольких IP адресов

sudo ip addr add 192.168.1.101/24 dev eth0
Добавляет дополнительный IP адрес на тот же интерфейс
📖 Пример: Настройка интерфейса с нуля 
# Отключаем интерфейс
sudo ip link set eth0 down

# Устанавливаем новый IP адрес
sudo ip addr add 192.168.1.50/24 dev eth0

# Включаем интерфейс
sudo ip link set eth0 up

# Добавляем шлюз по умолчанию
sudo ip route add default via 192.168.1.1

# Проверяем настройки
ip addr show eth0
ip route show
📖 Пример: Добавление нескольких IP адресов 
# Основной IP
sudo ip addr add 192.168.1.100/24 dev eth0

# Дополнительные IP адреса
sudo ip addr add 192.168.1.101/24 dev eth0
sudo ip addr add 192.168.1.102/24 dev eth0
sudo ip addr add 192.168.1.103/24 dev eth0

# Проверяем все адреса
ip addr show eth0

# Вывод покажет все 4 адреса на одном интерфейсе

Установка имени интерфейса (alias)

sudo ip link set eth0 alias "External Network"
Устанавливает описание интерфейса для удобства
💡 Преимущества команды ip:
  • Современный стандарт в Linux (заменяет ifconfig)
  • Больше функциональности (маршрутизация, туннели, политики)
  • Поддержка IPv6 из коробки
  • Более подробная информация об интерфейсах
  • Входит в пакет iproute2 (стандартно установлен)

⚙️ Команда ifconfig (устаревший метод)

⚠️ Внимание: ifconfig считается устаревшей командой. Рекомендуется использовать команду ip. Однако ifconfig всё ещё широко используется и может потребоваться для совместимости со старыми скриптами.

Установка ifconfig (если не установлен)

sudo apt install net-tools
Устанавливает пакет net-tools, содержащий ifconfig

Просмотр всех интерфейсов

ifconfig
Показывает все активные сетевые интерфейсы

Просмотр всех интерфейсов (включая неактивные)

ifconfig -a
Показывает все интерфейсы, даже выключенные

Просмотр конкретного интерфейса

ifconfig eth0
Показывает информацию только об eth0

Включение интерфейса

sudo ifconfig eth0 up
Активирует интерфейс eth0

Отключение интерфейса

sudo ifconfig eth0 down
Деактивирует интерфейс eth0

Назначение IP адреса

sudo ifconfig eth0 192.168.1.100 netmask 255.255.255.0
Устанавливает IP адрес и маску подсети

Изменение MTU

sudo ifconfig eth0 mtu 1400
Изменяет максимальный размер пакета

Изменение MAC адреса

sudo ifconfig eth0 hw ether 00:11:22:33:44:55
Изменяет MAC адрес интерфейса
💡 Таблица соответствия команд:
ifconfig (устарело) ip (современный) Описание
ifconfig ip addr show Показать все интерфейсы
ifconfig eth0 ip addr show eth0 Показать конкретный интерфейс
ifconfig eth0 up ip link set eth0 up Включить интерфейс
ifconfig eth0 down ip link set eth0 down Выключить интерфейс
ifconfig eth0 192.168.1.100 ip addr add 192.168.1.100/24 dev eth0 Назначить IP адрес
route ip route Показать таблицу маршрутизации
netstat ss Показать сетевые соединения
arp ip neigh Показать ARP таблицу

📝 Настройка интерфейсов через файлы

Основной файл конфигурации

sudo nano /etc/network/interfaces
Открывает главный файл конфигурации сетевых интерфейсов

Проверка конфигурации

cat /etc/network/interfaces
Просматривает текущую конфигурацию

Применение изменений (перезапуск интерфейса)

sudo ifdown eth0 && sudo ifup eth0
Перезапускает интерфейс для применения изменений

Перезапуск сетевой службы

sudo systemctl restart networking
Перезапускает службу networking (применяет все изменения)
📖 Пример: Базовая конфигурация /etc/network/interfaces 
# Loopback интерфейс
auto lo
iface lo inet loopback

# Основной Ethernet интерфейс с DHCP
auto eth0
iface eth0 inet dhcp

# WiFi интерфейс (управляется NetworkManager)
# Обычно не настраивается здесь
💡 Структура конфигурации:
  • auto eth0 - автоматически активировать при загрузке
  • allow-hotplug eth0 - активировать при обнаружении устройства
  • iface eth0 inet dhcp - использовать DHCP для IPv4
  • iface eth0 inet static - использовать статический IPv4
  • iface eth0 inet6 dhcp - использовать DHCP для IPv6
⚠️ Важно:
  • В Debian 11 с NetworkManager файл /etc/network/interfaces может игнорироваться для некоторых интерфейсов
  • Всегда делайте резервную копию перед изменениями: sudo cp /etc/network/interfaces /etc/network/interfaces.backup
  • Неправильная конфигурация может привести к потере сетевого подключения
  • Для серверов рекомендуется настройка через /etc/network/interfaces
  • Для десктопов обычно используется NetworkManager

🔢 Настройка статического IP адреса

Метод 1: Через файл конфигурации (постоянный)

sudo nano /etc/network/interfaces
Открывает файл для редактирования
📖 Пример: Конфигурация статического IP 
# Loopback
auto lo
iface lo inet loopback

# Статический IP на eth0
auto eth0
iface eth0 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 8.8.8.8 8.8.4.4
    dns-search example.com

# Или в формате CIDR
auto eth0
iface eth0 inet static
    address 192.168.1.100/24
    gateway 192.168.1.1
    dns-nameservers 8.8.8.8 8.8.4.4

Метод 2: Временная настройка (до перезагрузки)

sudo ip addr add 192.168.1.100/24 dev eth0
Добавляет IP адрес временно
sudo ip route add default via 192.168.1.1
Добавляет шлюз по умолчанию
📖 Пример: Полная настройка статического IP временно 
# Отключаем интерфейс
sudo ip link set eth0 down

# Очищаем старые настройки
sudo ip addr flush dev eth0

# Добавляем новый IP
sudo ip addr add 192.168.1.100/24 dev eth0

# Включаем интерфейс
sudo ip link set eth0 up

# Добавляем шлюз
sudo ip route add default via 192.168.1.1

# Настраиваем DNS (редактируем /etc/resolv.conf)
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
echo "nameserver 8.8.4.4" | sudo tee -a /etc/resolv.conf

# Проверяем
ip addr show eth0
ip route show
ping -c 3 8.8.8.8

Метод 3: Через nmcli (для NetworkManager)

nmcli connection modify eth0 ipv4.method manual ipv4.addresses 192.168.1.100/24 ipv4.gateway 192.168.1.1 ipv4.dns "8.8.8.8,8.8.4.4"
Настраивает статический IP через NetworkManager
nmcli connection up eth0
Применяет изменения
📖 Пример: Настройка нескольких статических IP 
# В файле /etc/network/interfaces

auto eth0
iface eth0 inet static
    address 192.168.1.100/24
    gateway 192.168.1.1

# Дополнительные IP адреса (aliases)
auto eth0:0
iface eth0:0 inet static
    address 192.168.1.101/24

auto eth0:1
iface eth0:1 inet static
    address 192.168.1.102/24

# Применяем
sudo systemctl restart networking
💡 Расчёт сетевых параметров:
CIDR Маска подсети Количество хостов Диапазон
/24 255.255.255.0 254 192.168.1.1 - 192.168.1.254
/16 255.255.0.0 65,534 192.168.0.1 - 192.168.255.254
/8 255.0.0.0 16,777,214 10.0.0.1 - 10.255.255.254
/30 255.255.255.252 2 Для point-to-point связей
/32 255.255.255.255 1 Один конкретный хост

🔄 DHCP настройка

Настройка DHCP через файл конфигурации

sudo nano /etc/network/interfaces
Открывает файл для настройки DHCP
📖 Пример: Конфигурация DHCP 
auto eth0
iface eth0 inet dhcp

# С дополнительными параметрами
auto eth0
iface eth0 inet dhcp
    hostname my-debian-pc
    client 00:11:22:33:44:55

Запрос нового IP от DHCP

sudo dhclient -r eth0
Освобождает текущий IP адрес
sudo dhclient eth0
Запрашивает новый IP адрес от DHCP сервера

Обновление DHCP аренды

sudo dhclient -v eth0
Обновляет аренду IP адреса с подробным выводом

Просмотр DHCP аренды

cat /var/lib/dhcp/dhclient.leases
Показывает информацию об арендованных IP адресах
📖 Пример: Освобождение и получение нового IP 
# Показать текущий IP
ip addr show eth0

# Освободить IP
sudo dhclient -r eth0

# Проверить - IP должен исчезнуть
ip addr show eth0

# Получить новый IP
sudo dhclient -v eth0

# Проверить новый IP
ip addr show eth0
ping -c 3 8.8.8.8
💡 DHCP клиенты в Debian:
  • dhclient - стандартный DHCP клиент ISC
  • dhcpcd - альтернативный DHCP клиент
  • NetworkManager - использует внутренний DHCP клиент

🛣️ Маршрутизация

Просмотр таблицы маршрутизации

ip route show
Показывает таблицу маршрутизации

Короткая форма

ip r
Краткая форма команды ip route show

Добавление шлюза по умолчанию

sudo ip route add default via 192.168.1.1
Добавляет маршрут по умолчанию через указанный шлюз

Удаление шлюза по умолчанию

sudo ip route del default
Удаляет маршрут по умолчанию

Добавление статического маршрута

sudo ip route add 10.0.0.0/24 via 192.168.1.254
Добавляет маршрут к сети 10.0.0.0/24 через шлюз 192.168.1.254

Удаление статического маршрута

sudo ip route del 10.0.0.0/24
Удаляет маршрут к указанной сети

Добавление маршрута через интерфейс

sudo ip route add 172.16.0.0/16 dev eth1
Добавляет маршрут к сети через конкретный интерфейс

Просмотр маршрута к конкретному хосту

ip route get 8.8.8.8
Показывает, через какой интерфейс и шлюз пройдёт пакет к 8.8.8.8
📖 Пример вывода ip route show: 
default via 192.168.1.1 dev eth0 proto static metric 100
10.0.0.0/24 via 192.168.1.254 dev eth0
172.16.0.0/16 dev eth1 proto kernel scope link src 172.16.1.100
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
Расшифровка:
  • default via 192.168.1.1 - маршрут по умолчанию через шлюз 192.168.1.1
  • dev eth0 - через интерфейс eth0
  • proto static - статически настроенный маршрут
  • proto kernel - автоматически созданный ядром
  • scope link - маршрут в локальной сети
  • src 192.168.1.100 - исходящий IP для этого маршрута
📖 Пример: Настройка маршрутизации между двумя сетями 
# Сценарий: компьютер с двумя сетевыми картами
# eth0: 192.168.1.100/24 (подключен к сети A)
# eth1: 10.0.0.1/24 (подключен к сети B)
# Задача: маршрутизировать трафик между сетями

# Включаем IP forwarding
sudo sysctl -w net.ipv4.ip_forward=1

# Делаем постоянным
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf

# На компьютерах в сети A (192.168.1.0/24):
# Добавляем маршрут к сети B через наш шлюз
sudo ip route add 10.0.0.0/24 via 192.168.1.100

# На компьютерах в сети B (10.0.0.0/24):
# Добавляем маршрут к сети A через наш шлюз
sudo ip route add 192.168.1.0/24 via 10.0.0.1

# Проверяем
ping -c 3 10.0.0.5  # с компьютера в сети A
📖 Пример: Постоянные маршруты в /etc/network/interfaces 
auto eth0
iface eth0 inet static
    address 192.168.1.100/24
    gateway 192.168.1.1
    # Добавляем статические маршруты
    up ip route add 10.0.0.0/24 via 192.168.1.254
    up ip route add 172.16.0.0/16 via 192.168.1.253
    # Удаляем при отключении
    down ip route del 10.0.0.0/24
    down ip route del 172.16.0.0/16

Очистка кэша маршрутизации

sudo ip route flush cache
Очищает кэш таблицы маршрутизации

Установка метрики маршрута

sudo ip route add default via 192.168.1.1 metric 100
Добавляет маршрут с приоритетом (меньше = выше приоритет)
💡 Понимание метрик:
  • Метрика определяет приоритет маршрута
  • Чем меньше метрика, тем выше приоритет
  • Используется при наличии нескольких маршрутов к одной сети
  • По умолчанию: проводные интерфейсы = 100, WiFi = 600

🌍 DNS настройка

Просмотр текущих DNS серверов

cat /etc/resolv.conf
Показывает настроенные DNS серверы

Временное изменение DNS

echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
Устанавливает Google DNS (изменения временные)

Добавление нескольких DNS серверов

sudo nano /etc/resolv.conf
Открывает файл для редактирования
📖 Пример: Содержимое /etc/resolv.conf 
# Google DNS
nameserver 8.8.8.8
nameserver 8.8.4.4

# Cloudflare DNS
nameserver 1.1.1.1
nameserver 1.0.0.1

# Локальный домен
search local.domain.com
domain local.domain.com

Постоянная настройка DNS (через interfaces)

sudo nano /etc/network/interfaces
Настройка DNS в конфигурации интерфейса
📖 Пример: DNS в /etc/network/interfaces 
auto eth0
iface eth0 inet static
    address 192.168.1.100/24
    gateway 192.168.1.1
    dns-nameservers 8.8.8.8 8.8.4.4
    dns-search example.com

Настройка через resolvconf

sudo apt install resolvconf
Устанавливает resolvconf для управления DNS
sudo nano /etc/resolvconf/resolv.conf.d/head
Редактирует постоянные DNS настройки
📖 Пример: Содержимое /etc/resolvconf/resolv.conf.d/head 
# Мои постоянные DNS серверы
nameserver 8.8.8.8
nameserver 8.8.4.4
nameserver 1.1.1.1

Обновление resolv.conf

sudo resolvconf -u
Обновляет /etc/resolv.conf на основе конфигурации

Проверка DNS резолвинга

nslookup google.com
Проверяет разрешение доменного имени

Подробная проверка DNS

dig google.com
Детальная информация о DNS запросе

Проверка с конкретным DNS сервером

dig @8.8.8.8 google.com
Запрашивает DNS у конкретного сервера

Краткий ответ DNS

dig +short google.com
Показывает только IP адрес

Обратный DNS lookup

dig -x 8.8.8.8
Определяет доменное имя по IP адресу

🔗 Bonding и VLAN

Установка необходимых пакетов

sudo apt install ifenslave vlan
Устанавливает инструменты для bonding и VLAN

Загрузка модуля bonding

Загружает модуль ядра для bonding
📖 Пример: Настройка bonding (объединение интерфейсов) 
# Создаём файл /etc/network/interfaces
auto bond0
iface bond0 inet static
    address 192.168.1.100/24
    gateway 192.168.1.1
    bond-slaves eth0 eth1

💡 Режимы bonding:
Режим Номер Описание
balance-rr 0 Round-robin, распределение нагрузки
active-backup 1 Один активный, остальные резервные
balance-xor 2 Распределение по MAC адресам
broadcast 3 Отправка на все интерфейсы
802.3ad 4 LACP агрегация (требует поддержки свитча)
balance-tlb 5 Адаптивная балансировка передачи
balance-alb 6 Адаптивная балансировка приёма/передачи

📖 Проблема 1: Нет подключения к интернету

ip addr show
# Проверяем наличие IP адреса

ping -c 3 192.168.1.1
# Проверяем доступность шлюза

# Обновляем IP от DHCP

# Проверяем маршруты

# Проверяем DNS

📖 Сценарий 1: Настройка домашнего сервера с двумя интерфейсами 
# eth0 - подключен к интернету (WAN)

iface eth0 inet dhcp

    address 192.168.100.1/24

# Включаем IP forwarding
sudo sysctl -w net.ipv4.ip_forward=1

# Настройка NAT (требуется iptables)
sudo apt install iptables-persistent
sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
sudo iptables -A FORWARD -i eth1 -o eth0 -j ACCEPT
sudo iptables -A FORWARD -i eth0 -o eth1 -m state --state RELATED,ESTABLISHED -j ACCEPT
sudo netfilter-persistent save

# Настройка DHCP сервера для LAN
sudo apt install isc-dhcp-server
# Настройте /etc/dhcp/dhcpd.conf

# wlan0 - WiFi адаптер

📖 Сценарий 3: Настройка резервного канала (failover) 
# eth0 - основной канал
# eth1 - резервный канал

# /etc/network/interfaces:
auto eth0
iface eth0 inet dhcp
    metric 100

auto eth1
iface eth1 inet dhcp
    metric 200

# Меньшая метрика = выше приоритет
# При падении eth0 автоматически переключится на eth1

# Проверка активного маршрута:
ip route show
# default via 192.168.1.1 dev eth0 metric 100
# default via 10.0.0.1 dev eth1 metric 200

📖 Сценарий 6: Скрипт для автоматической диагностики сети 
#!/bin/bash
# network-check.sh - Скрипт проверки сети

if [ -n "$GATEWAY" ]; then
    echo "Шлюз: $GATEWAY"
    ping -c 3 $GATEWAY
else
    echo "Шлюз не найден!"
fi

echo -e "\n=== Проверка внешней связи ==="
ping -c 3 8.8.8.8

echo -e "\n=== Проверка DNS ==="
ping -c 3 google.com

echo -e "\n=== Активные соединения ==="
ss -tunap | head -n 20

echo -e "\n=== Использование портов ==="
sudo ss -tulpn

echo -e "\n=== Диагностика завершена ==="
