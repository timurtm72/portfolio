📡 Решение проблемы с Wi-Fi на Orange Pi Zero 3

Полное руководство по устранению отключений Wi-Fi под Debian 11

📑 Содержание

❌ Описание проблемы 🔍 Диагностика 🔢 Статический IP 🤖 Watchdog скрипт 📡 Настройки роутера 🔌 Альтернативные решения

❌ Описание проблемы

🚨 Симптомы проблемы:
  • Wi-Fi периодически отключается через несколько минут после загрузки
  • Приходится вручную переподключаться командой nmcli connection up "ИМЯ_СОЕДИНЕНИЯ"
  • Проблема повторяется регулярно каждые 3-5 минут
  • В логах видны разрывы соединения во время 4-way handshake

Причина проблемы

Orange Pi Zero 3 использует встроенный Wi-Fi модуль Unisoc UWE5622, который имеет нестабильный драйвер под Linux. Драйвер не может корректно обрабатывать повторную аутентификацию WPA (4-way handshake), что приводит к регулярным разрывам соединения.

📖 Типичная запись в логах: 
device (wlan0): supplicant interface state: 4way_handshake -> disconnected
device (wlan0): Activation: (wifi) disconnected during association
💡 Техническая информация:
  • Платформа: Orange Pi Zero 3
  • ОС: Debian 11 (Bullseye)
  • Wi-Fi чип: Unisoc UWE5622
  • Прошивка: wifi_2355b001_1ant.ini
  • Тип проблемы: Нестабильность драйвера

🔍 Диагностика проблемы

Шаг 1: Установка необходимых утилит

sudo apt update && sudo apt install wireless-tools iw
Устанавливает утилиты для диагностики Wi-Fi

Шаг 2: Проверка энергосбережения

sudo iw dev wlan0 get power_save
Проверяет, включено ли энергосбережение (должно быть выключено)
💡 Если показывает "Power save: on":
sudo iw dev wlan0 set power_save off

Однако в нашем случае энергосбережение уже выключено, поэтому проблема не в этом.

Шаг 3: Проверка силы сигнала

sudo iw dev wlan0 link
Показывает информацию о текущем соединении
📖 Пример вывода (хороший сигнал): 
Connected to b0:fe:e5:c8:08:74 (on wlan0)
    SSID: HUAWEI-R1FUY4
    freq: 2412
    signal: -52 dBm  ← Отличный уровень сигнала
    tx bitrate: 52.0 MBit/s MCS 5
Уровень сигнала (dBm) Качество Описание
-30 до -50 Отлично Идеальный сигнал
-50 до -60 Хорошо Стабильное соединение
-60 до -70 Удовлетворительно Возможны небольшие задержки
-70 до -80 Слабо Нестабильное соединение
ниже -80 Очень слабо Постоянные разрывы

Шаг 4: Просмотр логов NetworkManager

sudo journalctl -u NetworkManager -n 50
Показывает последние 50 записей NetworkManager

Шаг 5: Мониторинг в реальном времени

sudo journalctl -u NetworkManager -f
Следит за логами в реальном времени (Ctrl+C для выхода)

Шаг 6: Проверка драйвера Wi-Fi

sudo dmesg | grep -E "wlan|wifi|unisoc"
Показывает информацию о драйвере Wi-Fi
📖 Вывод подтверждает использование Unisoc: 
wifi ini path = /lib/firmware/wifi_2355b001_1ant.ini
unisoc_wifi unisoc_wifi wlan0: mixed HW and IP checksum settings.

🔢 Попытка решения: Статический IP

Первая попытка - установка статического IP для исключения проблем с DHCP lease renewal.

Шаг 1: Определение текущих настроек

ip addr show wlan0
Показывает текущий IP адрес и маску сети
nmcli connection show
Показывает имена Wi-Fi подключений

Шаг 2: Настройка статического IP

sudo nmcli connection modify "HUAWEI-R1FUY4" ipv4.method manual ipv4.addresses 192.168.3.100/24 ipv4.gateway 192.168.3.1 ipv4.dns "8.8.8.8,8.8.4.4"
Устанавливает статический IP вместо DHCP
💡 Параметры команды:
  • ipv4.method manual - использовать статический IP (не DHCP)
  • ipv4.addresses 192.168.3.100/24 - ваш IP адрес и маска сети
  • /24 означает маску 255.255.255.0 (254 устройства в сети)
  • ipv4.gateway 192.168.3.1 - адрес роутера
  • ipv4.dns - DNS серверы (Google DNS)

Шаг 3: Применение изменений

sudo nmcli connection down "HUAWEI-R1FUY4"
Отключает соединение
sudo nmcli connection up "HUAWEI-R1FUY4"
Подключается с новыми настройками

Шаг 4: Проверка настроек

ip addr show wlan0
Проверяет, что IP установлен правильно
📖 Правильный вывод со статическим IP: 
13: wlan0:  mtu 1500
    inet 192.168.3.100/24 brd 192.168.3.255 scope global noprefixroute wlan0
       valid_lft forever preferred_lft forever  ← Статический IP
⚠️ Результат:

К сожалению, статический IP не решил проблему. Wi-Fi продолжает отключаться, потому что проблема именно в драйвере Unisoc, который некорректно обрабатывает повторную аутентификацию WPA.

Возврат к DHCP (если нужно)

sudo nmcli connection modify "HUAWEI-R1FUY4" ipv4.method auto
Возвращает настройки DHCP

🤖 Решение: Watchdog скрипт для автоматического переподключения

Поскольку проблема в драйвере и исправить её невозможно, создаём скрипт для автоматического обнаружения разрыва соединения и переподключения.

Шаг 1: Создание скрипта watchdog

sudo nano /usr/local/bin/wifi-watchdog.sh
Создаём файл скрипта
📖 Содержимое скрипта: 
#!/bin/bash
WIFI_INTERFACE="wlan0"
CONNECTION_NAME="HUAWEI-R1FUY4"
PING_TARGET="192.168.3.1"  # Адрес вашего роутера

while true; do
    # Проверяем связь с роутером
    if ! ping -c 1 -W 2 -I $WIFI_INTERFACE $PING_TARGET > /dev/null 2>&1; then
        logger "WiFi watchdog: Connection lost, reconnecting..."
        nmcli connection down "$CONNECTION_NAME" 2>/dev/null
        sleep 2
        nmcli connection up "$CONNECTION_NAME"
        sleep 5
    fi
    sleep 10  # Проверка каждые 10 секунд
done
💡 Как работает скрипт:
  • Каждые 10 секунд проверяет связь с роутером командой ping
  • Если ping не проходит - соединение разорвано
  • Отключает Wi-Fi соединение
  • Ждёт 2 секунды
  • Подключается заново
  • Записывает событие в системный лог

Шаг 2: Делаем скрипт исполняемым

sudo chmod +x /usr/local/bin/wifi-watchdog.sh
Даёт права на выполнение

Шаг 3: Создание systemd сервиса

sudo nano /etc/systemd/system/wifi-watchdog.service
Создаём файл сервиса
📖 Содержимое сервиса: 
[Unit]
Description=WiFi Watchdog
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/wifi-watchdog.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Шаг 4: Запуск и включение сервиса

sudo systemctl daemon-reload
Обновляет конфигурацию systemd
sudo systemctl enable wifi-watchdog
Включает автозапуск при загрузке
sudo systemctl start wifi-watchdog
Запускает сервис немедленно

Шаг 5: Проверка работы

sudo systemctl status wifi-watchdog
Проверяет статус сервиса
📖 Правильный вывод: 
● wifi-watchdog.service - WiFi Watchdog
     Loaded: loaded (/etc/systemd/system/wifi-watchdog.service; enabled)
     Active: active (running) since Sun 2025-10-12 10:05:44 UTC
   Main PID: 15499 (wifi-watchdog.s)
      Tasks: 2
     Memory: 1.4M

Просмотр логов watchdog

sudo journalctl -u wifi-watchdog -f
Следит за логами в реальном времени
sudo journalctl -u wifi-watchdog -n 20
Показывает последние 20 записей

Управление сервисом

sudo systemctl stop wifi-watchdog
Останавливает watchdog
sudo systemctl restart wifi-watchdog
Перезапускает watchdog
sudo systemctl disable wifi-watchdog
Отключает автозапуск
✅ Результат:

После установки watchdog скрипта Wi-Fi будет автоматически переподключаться в течение 10-12 секунд после разрыва. Вы больше не будете замечать отключений!

📡 Дополнительно: Настройки роутера

Можно попробовать изменить настройки роутера для уменьшения частоты переподключений.

Доступ к настройкам роутера

http://192.168.3.1
Обычный адрес веб-интерфейса роутера Huawei
💡 Рекомендуемые изменения в настройках Wi-Fi:

1. Режим безопасности

  • Измените с "WPA2/WPA3 Mixed" на "WPA2-PSK Only"
  • Шифрование: только AES (не TKIP, не AES+TKIP)
  • Отключите WPA3 если включён

2. Управление ключами

  • Group Key Update Interval - установите максимальное значение (3600 секунд или больше)
  • Это самая вероятная причина! Роутер обновляет групповой ключ, а драйвер не справляется

3. Дополнительные настройки

  • Отключите WPS (Wi-Fi Protected Setup)
  • Отключите 802.11w (Protected Management Frames) если есть
  • Отключите Fast Roaming (802.11r/k/v)
  • Отключите WMM (Wi-Fi Multimedia) как тест

4. Канал и частота

  • Используйте только 2.4 GHz (не 5 GHz для Orange Pi Zero 3)
  • Установите фиксированный канал: 1, 6 или 11
  • Отключите 802.11n 40MHz, используйте только 20MHz
📖 Оптимальная конфигурация Wi-Fi для Orange Pi: 
Режим: 802.11b/g/n mixed
Частота: 2.4 GHz only
Канал: 6 (фиксированный)
Ширина канала: 20 MHz
Безопасность: WPA2-PSK
Шифрование: AES
Group Key Update: 3600 секунд
WPS: Выключен
802.11w: Выключен
Fast Roaming: Выключен
⚠️ Важно:

После изменения настроек роутера необходимо переподключить все устройства заново, так как изменятся параметры безопасности.

🔌 Альтернативные решения

Решение 1: USB Wi-Fi адаптер (рекомендуется)

Самое надёжное решение - использовать внешний USB Wi-Fi адаптер с лучшим драйвером.

💡 Рекомендуемые чипы для Linux:
Чипсет Производитель Поддержка Linux
RTL8188EU Realtek Отличная, из коробки
RTL8192EU Realtek Отличная, из коробки
MT7601U MediaTek Отличная, стабильная
AR9271 Atheros Отличная, открытая

Решение 2: Отключение Bluetooth

Bluetooth и Wi-Fi используют общий чип и могут конфликтовать.

sudo systemctl stop bluetooth
Останавливает Bluetooth
sudo systemctl disable bluetooth
Отключает автозапуск Bluetooth

Решение 3: Обновление прошивки

sudo apt update && sudo apt install --reinstall linux-firmware-image-edge-sunxi64
Обновляет прошивку Wi-Fi модуля

Решение 4: Изменение интервала проверки watchdog

Если хотите более быструю реакцию на разрыв:

sudo nano /usr/local/bin/wifi-watchdog.sh
Редактируем скрипт

Измените строку sleep 10 в конце на sleep 5 для проверки каждые 5 секунд.

sudo systemctl restart wifi-watchdog
Перезапускаем с новыми настройками
⚠️ Компромисс:

Более частая проверка = быстрее обнаружение разрыва, но больше нагрузка на систему и батарею (если питание от аккумулятора).

✅ Итоговые рекомендации

🎯 Порядок действий для решения проблемы:
  1. Установить watchdog скрипт - основное и самое эффективное решение
  2. Настроить роутер - увеличить Group Key Update Interval до максимума
  3. Отключить Bluetooth - если не используется
  4. Купить USB Wi-Fi адаптер - если нужна максимальная стабильность
💡 Что делать, если watchdog не помогает:
  • Проверьте, что имя соединения в скрипте совпадает с реальным (nmcli connection show)
  • Проверьте адрес роутера (должен быть 192.168.3.1 или другой ваш шлюз)
  • Уменьшите интервал проверки с 10 до 5 секунд
  • Проверьте логи: sudo journalctl -u wifi-watchdog -n 50
