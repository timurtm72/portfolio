💻 Информация о системе

Общая информация о системе

uname -a
Показывает информацию о ядре, архитектуре, хостнейме

Версия ядра Linux

uname -r
Показывает только версию ядра

Архитектура системы

uname -m
Показывает архитектуру (x86_64, aarch64, и т.д.)

Информация о дистрибутиве

lsb_release -a
Показывает информацию о версии Debian

Альтернативный способ

cat /etc/os-release
Показывает детальную информацию о системе

Информация о hostname

hostname
Показывает имя компьютера

Изменение hostname

sudo hostnamectl set-hostname new-hostname
Устанавливает новое имя компьютера

Время работы системы (uptime)

uptime
Показывает, сколько времени система работает без перезагрузки

Дата последней загрузки

who -b
Показывает дату и время последней загрузки

Детальная информация о системе (dmidecode)

sudo dmidecode -t system
Показывает информацию о производителе, модели системы

Информация о BIOS

sudo dmidecode -t bios
Показывает информацию о BIOS/UEFI

Информация о материнской плате

sudo dmidecode -t baseboard
Показывает производителя и модель материнской платы
📖 Пример: Сбор полной информации о системе 
# Создаём скрипт system-info.sh
cat <<'EOF' > system-info.sh
#!/bin/bash
echo "=== Информация о системе ==="
echo "Hostname: $(hostname)"
echo "Дистрибутив: $(lsb_release -ds)"
echo "Ядро: $(uname -r)"
echo "Архитектура: $(uname -m)"
echo "Uptime: $(uptime -p)"
echo ""
echo "=== Производитель ==="
sudo dmidecode -s system-manufacturer
echo "Модель: $(sudo dmidecode -s system-product-name)"
echo "BIOS: $(sudo dmidecode -s bios-version)"
echo ""
echo "=== CPU ==="
lscpu | grep "Model name"
echo ""
echo "=== Память ==="
free -h | grep "Mem:"
echo ""
echo "=== Диски ==="
df -h | grep "^/dev"
EOF

chmod +x system-info.sh
./system-info.sh

Детальная информация через inxi

sudo apt install inxi
Устанавливает мощный инструмент для сбора информации
inxi -Fxz
Показывает полную информацию о системе
💡 Полезные опции dmidecode:
  • -t bios - информация о BIOS
  • -t system - информация о системе
  • -t baseboard - материнская плата
  • -t chassis - корпус
  • -t processor - процессор
  • -t memory - память
  • -t cache - кэш процессора

🔥 Процессор (CPU)

Информация о процессоре

lscpu
Показывает детальную информацию о CPU

Альтернативный способ

cat /proc/cpuinfo
Показывает информацию из ядра о каждом ядре процессора

Количество ядер

nproc
Показывает количество доступных ядер

Количество физических процессоров

grep "physical id" /proc/cpuinfo | sort -u | wc -l
Подсчитывает количество физических процессоров

Модель процессора

grep "model name" /proc/cpuinfo | head -1
Показывает модель CPU

Частота процессора

lscpu | grep MHz
Показывает текущую частоту всех ядер

Просмотр частоты в реальном времени

watch -n 1 "grep MHz /proc/cpuinfo"
Обновляет информацию о частоте каждую секунду

Управление частотой CPU (cpufreq)

sudo apt install cpufrequtils
Устанавливает утилиты управления частотой

Информация о cpufreq

cpufreq-info
Показывает информацию о политике управления частотой

Установка режима производительности

sudo cpufreq-set -g performance
Устанавливает максимальную производительность CPU

Режим энергосбережения

sudo cpufreq-set -g powersave
Устанавливает режим экономии энергии
💡 Режимы управления частотой (governors):
Режим Описание
performance Всегда максимальная частота
powersave Всегда минимальная частота
ondemand Динамическое изменение по нагрузке
conservative Плавное изменение частоты
schedutil Управление от планировщика ядра (современный)

Загрузка процессора

top
Интерактивный мониторинг процессов и загрузки CPU

Альтернатива - htop

sudo apt install htop
Устанавливает улучшенный top
htop
Запускает htop с цветным интерфейсом

Средняя загрузка (load average)

cat /proc/loadavg
Показывает среднюю загрузку за 1, 5 и 15 минут
📖 Пример: Тест производительности CPU 
# Установка sysbench
sudo apt install sysbench

# Однопоточный тест
sysbench cpu --cpu-max-prime=20000 run

# Многопоточный тест (все ядра)
sysbench cpu --cpu-max-prime=20000 --threads=$(nproc) run

# Стресс-тест CPU
sudo apt install stress
stress --cpu $(nproc) --timeout 60s

# Мониторинг во время теста
watch -n 1 "grep MHz /proc/cpuinfo | head -4"
📖 Пример: Ограничение CPU для процесса 
# Установка cpulimit
sudo apt install cpulimit

# Ограничить процесс до 50% CPU
cpulimit -p PID -l 50

# Запустить программу с ограничением
cpulimit -l 25 -- программа аргументы

# Найти PID процесса по имени
pidof имя_процесса

🧠 Память (RAM)

Использование памяти

free -h
Показывает использование RAM и swap в читаемом формате

Подробная информация

free -h --total
Показывает общую статистику с итогами

Обновление каждую секунду

watch -n 1 free -h
Мониторинг памяти в реальном времени

Информация о модулях памяти

sudo dmidecode -t memory
Показывает информацию о планках RAM

Краткая информация о памяти

sudo dmidecode -t memory | grep -E "Size|Speed|Manufacturer|Type:"
Фильтрует важную информацию о RAM

Максимальная поддерживаемая память

sudo dmidecode -t 16
Показывает максимальный объём поддерживаемой памяти

Количество слотов памяти

sudo dmidecode -t memory | grep "Number Of Devices"
Показывает количество слотов для RAM

Очистка кэша (освобождение памяти)

sudo sync && sudo sysctl -w vm.drop_caches=3
Очищает page cache, dentries и inodes
⚠️ Предупреждение: Очистка кэша обычно не требуется! Linux эффективно управляет памятью. Очистка может временно снизить производительность, так как придётся заново кэшировать данные.
💡 Понимание вывода free:
  • total - общий объём RAM
  • used - используется программами
  • free - совсем не используется
  • shared - используется tmpfs
  • buff/cache - используется для кэша (может быть освобождено)
  • available - доступно для программ (важнее чем free!)
📖 Пример вывода free -h: 
              total        used        free      shared  buff/cache   available
Mem:           15Gi       4.2Gi       8.1Gi       234Mi       3.2Gi        10Gi
Swap:         2.0Gi          0B       2.0Gi
Интерпретация: Из 16GB RAM используется 4.2GB программами, 3.2GB занято кэшем, доступно для программ 10GB. Swap не используется.

Топ процессов по памяти

ps aux --sort=-%mem | head -10
Показывает 10 процессов, потребляющих больше всего памяти

Информация о swap

swapon --show
Показывает информацию о swap разделах

Создание swap файла

sudo fallocate -l 2G /swapfile
Создаёт файл размером 2GB для swap
📖 Пример: Создание и активация swap файла 
# Создаём файл 4GB
sudo fallocate -l 4G /swapfile

# Устанавливаем правильные права
sudo chmod 600 /swapfile

# Форматируем как swap
sudo mkswap /swapfile

# Активируем
sudo swapon /swapfile

# Проверяем
swapon --show
free -h

# Делаем постоянным (добавляем в /etc/fstab)
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Настройка swappiness (насколько активно использовать swap)
# 0 = минимально, 100 = агрессивно, 10 = рекомендуется для desktop
sudo sysctl vm.swappiness=10
echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf

Удаление swap файла

sudo swapoff /swapfile && sudo rm /swapfile
Деактивирует и удаляет swap файл

Проверка утечек памяти

sudo apt install valgrind
Устанавливает инструмент для проверки утечек памяти

Список всех дисков

lsblk
Показывает все блочные устройства (диски, разделы)

Подробная информация

lsblk -f
Показывает файловые системы и UUID

Информация о дисках

sudo fdisk -l
Показывает все диски и их разделы

Использование дискового пространства

df -h
Показывает занятое и свободное место на разделах

Только локальные файловые системы

df -h -x tmpfs -x devtmpfs
Исключает временные файловые системы

Информация об inodes

df -i
Показывает использование inodes

Размер директории

du -sh /path/to/directory
Показывает общий размер директории

Топ директорий по размеру

du -h --max-depth=1 / 2>/dev/null | sort -rh | head -20
Показывает 20 самых больших директорий

S.M.A.R.T. информация о диске

sudo apt install smartmontools
Устанавливает утилиты S.M.A.R.T.

Проверка здоровья диска

sudo smartctl -a /dev/sda
Показывает S.M.A.R.T. данные диска

Быстрый тест диска

sudo smartctl -t short /dev/sda
Запускает короткий тест диска

Скорость чтения диска

sudo hdparm -Tt /dev/sda
Тестирует скорость чтения диска

Детальная информация о диске

sudo hdparm -I /dev/sda
Показывает детальную информацию о диске
📖 Пример: Тест производительности диска 
# Установка fio для тестирования
sudo apt install fio

# Тест случайного чтения (4K блоки)
sudo fio --name=random-read --ioengine=libaio --iodepth=32 \
  --rw=randread --bs=4k --direct=1 --size=1G --numjobs=1 \
  --runtime=60 --group_reporting --filename=/tmp/test-file

# Тест последовательной записи
sudo fio --name=sequential-write --ioengine=libaio --iodepth=1 \
  --rw=write --bs=1m --direct=1 --size=1G \
  --filename=/tmp/test-file

# Простой тест записи с dd
dd if=/dev/zero of=/tmp/test bs=1M count=1024 oflag=direct

# Простой тест чтения
dd if=/tmp/test of=/dev/null bs=1M

# Очистка
rm /tmp/test /tmp/test-file

Создание раздела (интерактивно)

sudo fdisk /dev/sdb
Запускает fdisk для создания разделов

Форматирование в ext4

sudo mkfs.ext4 /dev/sdb1
Форматирует раздел в ext4

Монтирование раздела

sudo mount /dev/sdb1 /mnt
Монтирует раздел в /mnt

Размонтирование

sudo umount /mnt
Размонтирует раздел

Просмотр смонтированных разделов

mount | grep "^/dev"
Показывает все смонтированные устройства
📖 Пример: Автомонтирование диска через /etc/fstab 
# Узнаём UUID диска
sudo blkid /dev/sdb1

# Например: UUID="1234-5678"

# Создаём точку монтирования
sudo mkdir /mnt/mydisk

# Редактируем /etc/fstab
sudo nano /etc/fstab

# Добавляем строку:
# UUID=1234-5678 /mnt/mydisk ext4 defaults 0 2

# Проверяем правильность fstab
sudo mount -a

# Если ошибок нет, диск смонтируется при следующей загрузке
💡 Типы файловых систем:
ФС Команда форматирования Применение
ext4 mkfs.ext4 Стандартная ФС Linux
xfs mkfs.xfs Для больших файлов
btrfs mkfs.btrfs Современная с снимками
ntfs mkfs.ntfs Совместимость с Windows
vfat mkfs.vfat USB флешки, SD карты
exfat mkfs.exfat Большие файлы на USB

🔌 USB устройства

Список USB устройств

lsusb
Показывает все подключенные USB устройства

Детальная информация

lsusb -v
Подробная информация о USB устройствах

Дерево USB устройств

lsusb -t
Показывает структуру USB портов и устройств

Информация об USB портах

sudo dmesg | grep -i usb
Показывает сообщения ядра об USB

Мониторинг подключения USB

sudo dmesg -w
Отслеживает подключение/отключение устройств

USB устройства в /dev

ls -l /dev/bus/usb/*/*
Показывает файлы USB устройств
📖 Пример: Определение USB флешки 
# Смотрим устройства до подключения
lsblk

# Подключаем флешку

# Смотрим что появилось
lsblk
# Допустим появился /dev/sdc1

# Получаем информацию
sudo fdisk -l /dev/sdc

# Проверяем файловую систему
sudo file -s /dev/sdc1

# Монтируем
sudo mount /dev/sdc1 /mnt

# Проверяем содержимое
ls /mnt

# Безопасно извлекаем
sudo umount /mnt
# Теперь можно физически отключить флешку

Форматирование USB флешки

sudo mkfs.vfat -F 32 /dev/sdc1
Форматирует в FAT32 (совместимость)

Форматирование в exFAT

sudo apt install exfat-fuse exfat-utils
Устанавливает поддержку exFAT
sudo mkfs.exfat /dev/sdc1
Форматирует в exFAT (для больших файлов)

Создание загрузочной USB флешки

sudo dd if=debian.iso of=/dev/sdc bs=4M status=progress && sync
Записывает ISO образ на флешку
⚠️ ВНИМАНИЕ: Команда dd полностью перезапишет USB флешку! Все данные будут потеряны. Убедитесь, что указали правильное устройство (/dev/sdX, не /dev/sdX1)!

Проверка скорости USB

sudo hdparm -tT /dev/sdc
Тестирует скорость чтения USB устройства
📖 Пример: Безопасное извлечение USB 
# Проверяем что смонтировано
mount | grep /dev/sdc1

# Размонтируем
sudo umount /dev/sdc1

# Проверяем что больше не смонтировано
mount | grep /dev/sdc1
# Должно быть пусто

# Опционально: очищаем буферы
sync

# Теперь можно физически извлечь флешку

#!/bin/bash
# temp-monitor.sh - Скрипт мониторинга температуры

TEMP_THRESHOLD=80  # Порог температуры в градусах
CHECK_INTERVAL=60  # Интервал проверки в секундах

while true; do
    # Получаем температуру CPU
    TEMP=$(sensors | grep "Package id 0:" | awk '{print $4}' | sed 's/+//;s/°C//')
    
    # Преобразуем в целое число
    TEMP_INT=${TEMP%.*}
    
    echo "$(date): Температура CPU: ${TEMP}°C"
    
    # Проверяем превышение порога
    if [ "$TEMP_INT" -gt "$TEMP_THRESHOLD" ]; then
        echo "ВНИМАНИЕ! Температура превысила ${TEMP_THRESHOLD}°C!"
        # Отправляем уведомление (требует notify-send)
        notify-send -u critical "Перегрев!" "CPU: ${TEMP}°C"
    fi
    
    sleep $CHECK_INTERVAL
done

#!/bin/bash
# battery-monitor.sh

while true; do
    CAPACITY=$(cat /sys/class/power_supply/BAT0/capacity)
    STATUS=$(cat /sys/class/power_supply/BAT0/status)
    
    echo "Заряд батареи: ${CAPACITY}% (${STATUS})"
    
    # Уведомление при низком заряде
    if [ "$CAPACITY" -lt 20 ] && [ "$STATUS" == "Discharging" ]; then
        notify-send -u critical "Низкий заряд!" "Батарея: ${CAPACITY}%"
    fi
    
    sleep 60
done

#!/bin/bash
# system-monitor.sh - Комплексный мониторинг

LOG_FILE="/var/log/system-monitor.log"

while true; do
    echo "=== $(date) ===" >> "$LOG_FILE"
    
    # CPU
    echo "CPU:" >> "$LOG_FILE"
    top -b -n1 | head -5 >> "$LOG_FILE"
    
    # Память
    echo -e "\nMemory:" >> "$LOG_FILE"
    free -h >> "$LOG_FILE"
    
    # Диски
    echo -e "\nDisks:" >> "$LOG_FILE"
    df -h | grep "^/dev" >> "$LOG_FILE"
    
    # Температура
    echo -e "\nTemperature:" >> "$LOG_FILE"
    sensors | grep -E "Core|Package" >> "$LOG_FILE"
    
    # Топ процессов по CPU
    echo -e "\nTop CPU:" >> "$LOG_FILE"
    ps aux --sort=-%cpu | head -6 >> "$LOG_FILE"
    
    # Топ процессов по RAM
    echo -e "\nTop RAM:" >> "$LOG_FILE"
    ps aux --sort=-%mem | head -6 >> "$LOG_FILE"
    
    echo -e "\n" >> "$LOG_FILE"
    
    sleep 300  # Каждые 5 минут
done
