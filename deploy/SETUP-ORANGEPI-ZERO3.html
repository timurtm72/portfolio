1.1Настройка приоритета WiFi (опционально)

⚠️ Важно! Если у вас подключены и WiFi и Ethernet одновременно, нужно настроить приоритет. Это сохранится после перезагрузки и будет работать с любым роутером.

Ситуация: WiFi в приоритете, Ethernet как резерв

Это нужно если вы хотите использовать WiFi как основной интернет, а Ethernet как резервный канал.

# 1. Проверить текущие маршруты
ip route show | grep default

# Вы увидите что-то вроде:
# default via 192.168.1.1 dev eth0
# default via 172.16.16.1 dev wlan0 proto dhcp metric 600

Настройка приоритета через метрику

Меньше метрика = выше приоритет. WiFi ставим 100, Ethernet 200:

# 2. Изменить приоритет WiFi на высокий (метрика 100)
sudo nmcli connection modify "ИМЯ_ВАШЕЙ_WIFI" ipv4.route-metric 100

# Например для сети "montanum-2G":
# sudo nmcli connection modify "montanum-2G" ipv4.route-metric 100

# Перезапустить WiFi соединение
sudo nmcli connection down "ИМЯ_ВАШЕЙ_WIFI"
sudo nmcli connection up "ИМЯ_ВАШЕЙ_WIFI"

Настройка Ethernet с низким приоритетом (постоянно)

# 3. Редактировать конфигурацию Ethernet
sudo nano /etc/network/interfaces

# Найдите строки про eth0 и добавьте метрику:
auto eth0
iface eth0 inet dhcp
    metric 200

# Сохранить: Ctrl+O, Enter
# Выйти: Ctrl+X

# 4. Перезагрузить сеть
sudo systemctl restart networking

# Или перезагрузить плату
sudo reboot

Проверка после перезагрузки

# После перезагрузки проверить маршруты
ip route show | grep default

# Должно быть:
# default via 172.16.16.1 dev wlan0 proto dhcp metric 100  ← WiFi первый
# default via 192.168.1.1 dev eth0 metric 200             ← Ethernet резервный

# Проверить интернет
ping -c 4 8.8.8.8
ping -c 4 google.com
✅ Готово! Теперь Orange Pi всегда будет использовать WiFi первым, а Ethernet только если WiFi недоступен. Настройки сохранятся после перезагрузки и будут работать с любым роутером!

Смена WiFi сети

Когда подключаетесь к новому роутеру вечером:

# Посмотреть доступные сети
nmcli device wifi list

# Подключиться к новой сети
sudo nmcli device wifi connect "НОВАЯ_СЕТЬ" password "ПАРОЛЬ"

# Установить высокий приоритет для новой сети
sudo nmcli connection modify "НОВАЯ_СЕТЬ" ipv4.route-metric 100

# Перезапустить соединение
sudo nmcli connection down "НОВАЯ_СЕТЬ"
sudo nmcli connection up "НОВАЯ_СЕТЬ"

# Проверить
ip route show | grep default
💡 Совет: Для сервера рекомендуется использовать Ethernet (более стабильное соединение). WiFi лучше как резервный канал. Но если нужен именно WiFi - используйте эти настройки.

1.2Автоподключение к нескольким WiFi сетям

🎯 Сценарий использования

Если вы перемещаете Orange Pi между разными местами (дом, работа, дача), настройте автоматическое подключение ко всем вашим WiFi сетям. При загрузке плата автоматически подключится к доступной сети с наивысшим приоритетом.

Добавление нескольких WiFi сетей с приоритетами

# 1. Первая сеть (домашняя) - наивысший приоритет 100
sudo nmcli connection add type wifi ifname wlan0 \
  con-name "WiFi-Home" \
  ssid "montanum-2G" \
  wifi-sec.key-mgmt wpa-psk \
  wifi-sec.psk "ВАШ_ПАРОЛЬ"

sudo nmcli connection modify "WiFi-Home" \
  connection.autoconnect yes \
  connection.autoconnect-priority 100 \
  ipv4.route-metric 100

# 2. Вторая сеть (вечером/другое место) - приоритет 90
sudo nmcli connection add type wifi ifname wlan0 \
  con-name "WiFi-Evening" \
  ssid "НАЗВАНИЕ_ВЕЧЕРНЕЙ_СЕТИ" \
  wifi-sec.key-mgmt wpa-psk \
  wifi-sec.psk "ПАРОЛЬ_ВЕЧЕРНЕЙ_СЕТИ"

sudo nmcli connection modify "WiFi-Evening" \
  connection.autoconnect yes \
  connection.autoconnect-priority 90 \
  ipv4.route-metric 100

# 3. Третья сеть (работа/дача) - приоритет 80
sudo nmcli connection add type wifi ifname wlan0 \
  con-name "WiFi-Work" \
  ssid "РАБОЧАЯ_СЕТЬ" \
  wifi-sec.key-mgmt wpa-psk \
  wifi-sec.psk "ПАРОЛЬ_РАБОТЫ"

sudo nmcli connection modify "WiFi-Work" \
  connection.autoconnect yes \
  connection.autoconnect-priority 80 \
  ipv4.route-metric 100

Настройка Ethernet как резервного канала

# Редактировать конфигурацию Ethernet
sudo nano /etc/network/interfaces

# Добавить метрику для eth0:
auto eth0
iface eth0 inet dhcp
    metric 200

# Сохранить: Ctrl+O, Enter
# Выйти: Ctrl+X

Проверка настроек

# Посмотреть все сохраненные подключения
nmcli connection show

# Вы увидите:
# NAME          UUID                                  TYPE      DEVICE
# WiFi-Home     xxxx-xxxx-xxxx-xxxx                  wifi      wlan0
# WiFi-Evening  yyyy-yyyy-yyyy-yyyy                  wifi      --
# WiFi-Work     zzzz-zzzz-zzzz-zzzz                  wifi      --

# Перезагрузить для проверки автоподключения
sudo reboot

Как это работает после перезагрузки

✅ Автоматическое поведение:
  1. Orange Pi сканирует доступные WiFi сети
  2. Подключается к сети с наивысшим приоритетом (100 > 90 > 80)
  3. Если сеть с приоритетом 100 недоступна, подключится к следующей
  4. WiFi всегда используется первым (метрика 100)
  5. Ethernet включается только если WiFi не работает (метрика 200)

Добавление новой WiFi сети в новом месте

Когда окажетесь в новом месте с другим WiFi:

# 1. Посмотреть доступные сети
nmcli device wifi list

# 2. Подключиться к новой сети
sudo nmcli device wifi connect "НОВАЯ_СЕТЬ" password "ПАРОЛЬ"

# 3. Настроить автоподключение и приоритет
# Приоритет 85 = между домашней (100) и вечерней (90)
sudo nmcli connection modify "НОВАЯ_СЕТЬ" \
  connection.autoconnect yes \
  connection.autoconnect-priority 85 \
  ipv4.route-metric 100

# 4. Проверить все подключения
nmcli connection show

# 5. Проверить что интернет работает
ping -c 4 google.com

Удаление ненужной WiFi сети

# Посмотреть все сохраненные сети
nmcli connection show

# Удалить ненужную сеть
sudo nmcli connection delete "WiFi-Old-Network"

Изменение приоритета существующей сети

# Повысить приоритет сети (чем больше число, тем выше приоритет)
sudo nmcli connection modify "WiFi-Work" connection.autoconnect-priority 95

# Понизить приоритет
sudo nmcli connection modify "WiFi-Home" connection.autoconnect-priority 70
⚠️ Важно:
  • Все пароли сохраняются в системе - будьте аккуратны с безопасностью
  • Приоритет: большее число = выше приоритет (100 > 90 > 80)
  • Метрика маршрута: меньшее число = выше приоритет (100 < 200)
  • Настройки сохраняются после перезагрузки автоматически

Проверка текущего подключения

# Узнать к какой сети подключен сейчас
nmcli connection show --active

# Посмотреть маршруты
ip route show | grep default

# Проверить интернет
ping -c 4 8.8.8.8
ping -c 4 google.com
🎉 Готово! Теперь Orange Pi автоматически подключится к любой из ваших сохраненных WiFi сетей при загрузке. Берите плату куда угодно - она сама найдет знакомую сеть!

2Установка Docker

2.1. Установка Docker Engine

# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка зависимостей
sudo apt install -y ca-certificates curl gnupg

# Добавление GPG ключа Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Добавление репозитория Docker
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Установка Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Проверка установки
docker --version
docker compose version

2.2. Настройка прав

# Добавить пользователя в группу docker
sudo usermod -aG docker $USER

# Перезайти по SSH для применения прав
exit

# Снова подключиться
ssh orangepi@192.168.1.100

# Проверка (должно работать без sudo)
docker ps

3Настройка SWAP (для 1GB RAM)

⚠️ Только для версий с 1GB RAM! Если у вас 2GB+ RAM, пропустите этот шаг. 
# Создание swap файла 2GB
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Автозагрузка при перезагрузке
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Проверка
free -h

4Клонирование проекта с GitHub

4.1. Установка git

sudo apt install -y git

4.2. Клонирование проекта

📦 Выбор метода клонирования

GitHub требует аутентификацию. Есть два способа:

Способ 1: Публичный репозиторий (самый простой)

Если репозиторий публичный, просто клонируйте:

cd ~
git clone https://github.com/timurtm72/portfolio.git
cd portfolio
ls -la
⚠️ Если появляется ошибка "Authentication failed", значит репозиторий приватный. Используйте Способ 2.

Способ 2: Приватный репозиторий (через Personal Access Token)

Шаг 1: Создайте токен на GitHub (на вашем компьютере):

  1. Откройте: https://github.com/settings/tokens
  2. Нажмите "Generate new token" → "Generate new token (classic)"
  3. Дайте имя: Orange Pi Deploy
  4. Выберите срок: 90 days или No expiration
  5. Отметьте галочки:
    • ✅ repo (все подпункты!)
  6. Нажмите "Generate token"
  7. ВАЖНО: Скопируйте токен (выглядит как ghp_xxxxxxxxxxxx)

Шаг 2: Клонируйте на Orange Pi с токеном:

cd ~
git clone https://ВАШ_ТОКЕН@github.com/timurtm72/portfolio.git
cd portfolio

# Сохранить токен для будущих git pull
git config --global credential.helper store

# Проверка файлов
ls -la
🔐 ВАЖНО - Безопасность токена:
  • НИКОГДА не публикуйте токен в чатах, форумах или публичных местах
  • Токен дает полный доступ к вашим репозиториям
  • Если токен скомпрометирован - немедленно удалите его на GitHub
  • Храните токен в безопасном месте (менеджер паролей)

Способ 3: Сделать репозиторий публичным

Если репозиторий приватный и вы хотите сделать его публичным:

  1. Откройте: https://github.com/timurtm72/portfolio/settings
  2. Прокрутите вниз до "Danger Zone"
  3. Нажмите "Change visibility" → "Make public"
  4. Подтвердите

После этого используйте Способ 1 (без токена).

7Настройка доступа из интернета

7.1. Узнать локальный IP Orange Pi

ip addr show | grep "inet " | grep -v "127.0.0.1"

# Например: 192.168.1.100

7.2. Настроить роутер

Зайдите в веб-интерфейс вашего роутера (обычно 192.168.1.1 или 192.168.0.1) и настройте проброс портов (Port Forwarding):

Название Внешний порт Внутренний IP Внутренний порт Протокол
Portfolio Web 80 192.168.1.100 3000 TCP
Portfolio API 8080 192.168.1.100 8080 TCP
⚠️ Примечание: Некоторые провайдеры блокируют порт 80. В этом случае используйте другой порт, например 8080 → 3000.

7.3. Узнать внешний IP адрес

# На Orange Pi
curl ifconfig.me

# Или на компьютере перейти на сайт:
# https://whatismyipaddress.com/

7.4. Тестирование

# С другого устройства в интернете
curl http://ВАШ_ВНЕШНИЙ_IP

# Или откройте в браузере:
http://ВАШ_ВНЕШНИЙ_IP
