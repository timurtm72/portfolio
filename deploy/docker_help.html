⚙️ Установка Docker

Шаг 1: Обновление системы

sudo apt update && sudo apt upgrade -y
Обновляет список пакетов и саму систему

Шаг 2: Установка необходимых пакетов

sudo apt install apt-transport-https ca-certificates curl gnupg lsb-release -y
Устанавливает зависимости для добавления репозитория Docker

Шаг 3: Добавление GPG ключа Docker

curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
Загружает и устанавливает официальный GPG ключ Docker

Шаг 4: Добавление репозитория Docker

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
Добавляет официальный репозиторий Docker в систему

Шаг 5: Установка Docker Engine

sudo apt update && sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin -y
Устанавливает Docker Engine, CLI и Docker Compose

Шаг 6: Добавление пользователя в группу docker

sudo usermod -aG docker $USER
Позволяет запускать Docker без sudo (требуется перелогиниться)

Проверка установки

docker --version
Показывает версию установленного Docker

Запуск тестового контейнера

docker run hello-world
Проверяет работоспособность Docker
💡 Важно: После добавления пользователя в группу docker необходимо выйти из системы и войти заново, чтобы изменения вступили в силу.

🚀 Основные команды

Проверка версии Docker

docker version
Показывает детальную информацию о версии Docker (клиент и сервер)

Информация о системе Docker

docker info
Выводит системную информацию Docker (количество контейнеров, образов, версия ядра и т.д.)

Справка по командам

docker --help
Показывает список всех доступных команд Docker

Справка по конкретной команде

docker run --help
Показывает справку по конкретной команде (например, run)

Проверка статуса Docker службы

sudo systemctl status docker
Проверяет, запущена ли служба Docker

Запуск службы Docker

sudo systemctl start docker
Запускает службу Docker

Остановка службы Docker

sudo systemctl stop docker
Останавливает службу Docker

Перезапуск службы Docker

sudo systemctl restart docker
Перезапускает службу Docker

Автозапуск Docker при загрузке системы

sudo systemctl enable docker
Включает автоматический запуск Docker при старте системы

💿 Работа с образами (Images)

Поиск образов на Docker Hub

docker search nginx
Ищет образы nginx в Docker Hub

Скачивание образа

docker pull nginx
Загружает последнюю версию образа nginx

Скачивание конкретной версии

docker pull nginx:1.21
Загружает конкретную версию образа (тег 1.21)

Список локальных образов

docker images
Показывает все образы на локальной машине

Список образов с форматированием

docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"
Выводит образы в читаемом формате с конкретными столбцами

Детальная информация об образе

docker inspect nginx
Показывает подробную информацию об образе в формате JSON

История образа

docker history nginx
Показывает историю слоёв образа

Удаление образа

docker rmi nginx
Удаляет образ nginx (должен быть неиспользуемым)

Принудительное удаление образа

docker rmi -f nginx
Принудительно удаляет образ, даже если он используется

Удаление нескольких образов

docker rmi nginx mysql redis
Удаляет несколько образов одновременно

Создание образа из контейнера

docker commit контейнер_id новый_образ:тег
Создаёт новый образ из изменённого контейнера

Сохранение образа в файл

docker save -o nginx.tar nginx
Экспортирует образ в tar-архив

Загрузка образа из файла

docker load -i nginx.tar
Импортирует образ из tar-архива

Тегирование образа

docker tag nginx:latest myrepo/nginx:v1
Создаёт новый тег для образа
📖 Пример: Скачивание и использование образа Ubuntu 
# Скачиваем образ Ubuntu 20.04
docker pull ubuntu:20.04

# Проверяем, что образ загружен
docker images | grep ubuntu

# Запускаем контейнер на основе образа
docker run -it ubuntu:20.04 bash

📦 Работа с контейнерами (Containers)

Запуск контейнера

docker run nginx
Создаёт и запускает контейнер из образа nginx

Запуск в фоновом режиме (detached)

docker run -d nginx
Запускает контейнер в фоновом режиме

Запуск с именем контейнера

docker run -d --name my-nginx nginx
Запускает контейнер с пользовательским именем

Запуск с пробросом портов

docker run -d -p 8080:80 nginx
Пробрасывает порт 80 контейнера на порт 8080 хоста

Запуск с переменными окружения

docker run -d -e MYSQL_ROOT_PASSWORD=secret mysql
Устанавливает переменные окружения для контейнера

Запуск с монтированием тома

docker run -d -v /host/path:/container/path nginx
Монтирует локальную директорию в контейнер

Запуск с автоматическим удалением

docker run --rm -it ubuntu bash
Автоматически удаляет контейнер после остановки

Запуск с ограничением ресурсов

docker run -d --memory="512m" --cpus="1.5" nginx
Ограничивает использование памяти (512MB) и CPU (1.5 ядра)

Список запущенных контейнеров

docker ps
Показывает только активные контейнеры

Список всех контейнеров

docker ps -a
Показывает все контейнеры (включая остановленные)

Список контейнеров с размерами

docker ps -as
Показывает контейнеры с информацией о занимаемом месте

Остановка контейнера

docker stop my-nginx
Корректно останавливает контейнер (даёт 10 секунд на завершение)

Принудительная остановка

docker kill my-nginx
Немедленно убивает процесс контейнера

Запуск остановленного контейнера

docker start my-nginx
Запускает ранее остановленный контейнер

Перезапуск контейнера

docker restart my-nginx
Останавливает и снова запускает контейнер

Пауза контейнера

docker pause my-nginx
Приостанавливает все процессы в контейнере

Снятие с паузы

docker unpause my-nginx
Возобновляет работу приостановленного контейнера

Удаление контейнера

docker rm my-nginx
Удаляет остановленный контейнер

Принудительное удаление работающего контейнера

docker rm -f my-nginx
Останавливает и удаляет контейнер

Выполнение команды в контейнере

docker exec my-nginx ls /etc
Выполняет команду в работающем контейнере

Подключение к терминалу контейнера

docker exec -it my-nginx bash
Открывает интерактивный терминал внутри контейнера

Подключение к запущенному контейнеру

docker attach my-nginx
Подключается к stdin/stdout работающего контейнера

Копирование файлов из контейнера

docker cp my-nginx:/etc/nginx/nginx.conf ./nginx.conf
Копирует файл из контейнера на хост

Копирование файлов в контейнер

docker cp ./index.html my-nginx:/usr/share/nginx/html/
Копирует файл с хоста в контейнер

Переименование контейнера

docker rename old-name new-name
Изменяет имя контейнера

Информация о контейнере

docker inspect my-nginx
Показывает детальную информацию о контейнере в формате JSON

Статистика использования ресурсов

docker stats my-nginx
Показывает использование CPU, памяти, сети в реальном времени

Процессы внутри контейнера

docker top my-nginx
Показывает запущенные процессы в контейнере

Изменения в файловой системе

docker diff my-nginx
Показывает изменения в файловой системе контейнера

Экспорт контейнера

docker export my-nginx > nginx-container.tar
Экспортирует файловую систему контейнера в tar-архив

# Проверяем, что контейнер запущен

🌐 Сети Docker

Список сетей

docker network ls
Показывает все Docker-сети

Создание сети

docker network create my-network
Создаёт новую bridge-сеть

Создание сети с подсетью

docker network create --subnet=172.20.0.0/16 my-network
Создаёт сеть с указанной подсетью

Информация о сети

docker network inspect my-network
Показывает детальную информацию о сети

Подключение контейнера к сети

docker network connect my-network my-nginx
Подключает существующий контейнер к сети

Отключение контейнера от сети

docker network disconnect my-network my-nginx
Отключает контейнер от сети

Удаление сети

docker network rm my-network
Удаляет сеть (должна быть неиспользуемой)

Удаление всех неиспользуемых сетей

docker network prune
Удаляет все сети без подключенных контейнеров
💡 Типы сетей Docker:
  • bridge - стандартная сеть, контейнеры могут общаться между собой
  • host - контейнер использует сеть хоста напрямую
  • none - контейнер не имеет сетевого доступа
  • overlay - для связи контейнеров на разных хостах (Docker Swarm)
📖 Пример: Создание сети и запуск контейнеров 
# Создаём пользовательскую сеть
docker network create app-network

# Запускаем MySQL в этой сети
docker run -d --name mysql-db \
  --network app-network \
  -e MYSQL_ROOT_PASSWORD=secret \
  mysql:8.0

# Запускаем WordPress в той же сети
docker run -d --name wordpress \
  --network app-network \
  -p 80:80 \
  -e WORDPRESS_DB_HOST=mysql-db \
  -e WORDPRESS_DB_PASSWORD=secret \
  wordpress

# Контейнеры могут общаться по именам!

💾 Тома (Volumes)

Список томов

docker volume ls
Показывает все Docker-тома

Создание тома

docker volume create my-volume
Создаёт именованный том

Информация о томе

docker volume inspect my-volume
Показывает детальную информацию о томе

Удаление тома

docker volume rm my-volume
Удаляет том (должен быть неиспользуемым)

Удаление всех неиспользуемых томов

docker volume prune
Удаляет все тома без подключенных контейнеров

Использование тома при запуске

docker run -d -v my-volume:/app/data nginx
Монтирует том в контейнер

Монтирование директории хоста (bind mount)

docker run -d -v /home/user/data:/app/data nginx
Монтирует директорию хоста в контейнер

Монтирование только для чтения

docker run -d -v my-volume:/app/data:ro nginx
Монтирует том в режиме только для чтения
💡 Различия между Volume и Bind Mount:
  • Volume - управляется Docker, хранится в /var/lib/docker/volumes/
  • Bind Mount - прямое монтирование директории хоста, полный контроль над путём
  • Volumes рекомендуются для продакшена, проще резервировать и мигрировать
📖 Пример: Постоянное хранение данных PostgreSQL 
# Создаём том для БД
docker volume create postgres-data

# Запускаем PostgreSQL с томом
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=mypassword \
  -v postgres-data:/var/lib/postgresql/data \
  -p 5432:5432 \
  postgres:14

# Данные будут сохранены даже после удаления контейнера
docker rm -f postgres

# Запускаем новый контейнер с тем же томом
docker run -d \
  --name postgres-new \
  -e POSTGRES_PASSWORD=mypassword \
  -v postgres-data:/var/lib/postgresql/data \
  -p 5432:5432 \
  postgres:14

# Все данные сохранились!

📝 Dockerfile

Создание образа из Dockerfile

docker build -t my-image:v1 .
Собирает образ из Dockerfile в текущей директории

Создание с указанием Dockerfile

docker build -t my-image -f Dockerfile.prod .
Использует конкретный Dockerfile

Сборка без кэша

docker build --no-cache -t my-image .
Собирает образ без использования кэша слоёв

Сборка с аргументами

docker build --build-arg VERSION=1.0 -t my-image .
Передаёт аргументы в Dockerfile (через ARG)
💡 Основные инструкции Dockerfile:
Инструкция Описание Пример
FROM Базовый образ FROM ubuntu:20.04
RUN Выполняет команду при сборке RUN apt-get update
CMD Команда по умолчанию при запуске CMD ["nginx", "-g", "daemon off;"]
ENTRYPOINT Точка входа (не перезаписывается) ENTRYPOINT ["python"]
COPY Копирует файлы в образ COPY . /app
ADD Копирует файлы (с доп. функциями) ADD archive.tar.gz /app
WORKDIR Устанавливает рабочую директорию WORKDIR /app
ENV Устанавливает переменную окружения ENV NODE_ENV=production
EXPOSE Указывает порт EXPOSE 80
VOLUME Создаёт точку монтирования VOLUME /data
USER Устанавливает пользователя USER nginx
ARG Аргумент при сборке ARG VERSION=1.0
📖 Пример: Простой Dockerfile для Node.js приложения 
# Используем официальный образ Node.js
FROM node:16-alpine

# Устанавливаем рабочую директорию
WORKDIR /app

# Копируем package.json и package-lock.json
COPY package*.json ./

# Устанавливаем зависимости
RUN npm ci --only=production

# Копируем остальные файлы приложения
COPY . .

# Указываем порт
EXPOSE 3000

# Запускаем приложение
CMD ["node", "server.js"]
📖 Пример: Multi-stage сборка для Go приложения 
# Стадия сборки
FROM golang:1.19-alpine AS builder
WORKDIR /build
COPY . .
RUN go build -o app main.go

# Финальный образ
FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /build/app .
EXPOSE 8080
CMD ["./app"]
📖 Пример: Dockerfile для Python Flask приложения 
# Базовый образ Python
FROM python:3.9-slim

# Устанавливаем рабочую директорию
WORKDIR /app

# Копируем requirements.txt
COPY requirements.txt .

# Устанавливаем зависимости
RUN pip install --no-cache-dir -r requirements.txt

# Копируем код приложения
COPY . .

# Создаём пользователя без root-прав
RUN useradd -m appuser && chown -R appuser /app
USER appuser

# Указываем порт
EXPOSE 5000

# Переменная окружения
ENV FLASK_APP=app.py

# Запускаем приложение
CMD ["flask", "run", "--host=0.0.0.0"]
⚠️ Best Practices для Dockerfile:
  • Используйте .dockerignore для исключения ненужных файлов
  • Объединяйте RUN команды для уменьшения слоёв
  • Копируйте только необходимые файлы
  • Используйте multi-stage сборку для уменьшения размера образа
  • Не запускайте процессы от root (используйте USER)
  • Сортируйте apt-get install для лучшего кэширования

Запуск с пересборкой образов

docker compose up --build

Остановка сервисов

docker compose down
Останавливает и удаляет контейнеры, сети

Остановка с удалением томов

docker compose down -v
Удаляет контейнеры, сети И тома

Запускает команду в новом контейнере

Остановка сервисов без удаления

🏪 Docker Hub & Registry

Вход в Docker Hub

docker login
Авторизует в Docker Hub (запросит логин и пароль)

Вход с указанием логина

docker login -u username
Авторизует с конкретным именем пользователя

Выход из Docker Hub

docker logout
Удаляет сохранённые учётные данные

Отправка образа в Docker Hub

docker push username/image:tag
Загружает образ в Docker Hub

Скачивание образа из Docker Hub

docker pull username/image:tag
Загружает образ из Docker Hub
📖 Пример: Публикация образа в Docker Hub 
# 1. Создайте образ
docker build -t my-app .

# 2. Тегируйте образ с вашим именем пользователя
docker tag my-app username/my-app:v1.0

# 3. Войдите в Docker Hub
docker login

# 4. Отправьте образ
docker push username/my-app:v1.0

# 5. Теперь любой может скачать ваш образ
docker pull username/my-app:v1.0

Авторизует в приватном registry

Отправка в приватный registry

Загружает образ в приватный registry

docker logs -f my-nginx
Следит за логами в режиме реального времени

Последние N строк логов

docker logs --tail 100 my-nginx
Показывает последние 100 строк логов

Логи с временными метками

docker logs -t my-nginx
Добавляет временные метки к каждой строке лога

Логи за определённое время

docker logs --since 1h my-nginx
Показывает логи за последний час

🧹 Очистка системы

Удаление остановленных контейнеров

docker container prune
Удаляет все остановленные контейнеры

Удаление неиспользуемых образов

docker image prune
Удаляет "висящие" образы (без тегов)

Удаление всех неиспользуемых образов

docker image prune -a
Удаляет все образы, не используемые контейнерами

Удаление неиспользуемых томов

docker volume prune
Удаляет все тома без подключенных контейнеров

Удаление неиспользуемых сетей

docker network prune
Удаляет все сети без подключенных контейнеров

Полная очистка системы

docker system prune
Удаляет остановленные контейнеры, неиспользуемые сети, висящие образы

Агрессивная очистка

docker system prune -a --volumes
Удаляет ВСЁ неиспользуемое (контейнеры, образы, сети, тома)

Удаление всех контейнеров

docker rm -f $(docker ps -aq)
Принудительно удаляет абсолютно все контейнеры

Удаление всех образов

docker rmi -f $(docker images -q)
Принудительно удаляет все образы
⚠️ ВНИМАНИЕ: Команды полной очистки удаляют данные безвозвратно! Убедитесь, что у вас есть резервные копии важных данных перед использованием этих команд.
📖 Пример: Безопасная очистка 
# 1. Проверьте, что занимает место
docker system df -v

# 2. Удалите остановленные контейнеры
docker container prune

# 3. Удалите неиспользуемые образы
docker image prune

# 4. Удалите неиспользуемые тома
docker volume prune

# 5. Проверьте результат
docker system df

📖 Пример 2: Создание базы данных PostgreSQL 
# Создайте том для данных
docker volume create pgdata

# Запустите PostgreSQL
docker run -d \
  --name postgres-db \
  -e POSTGRES_PASSWORD=mysecretpassword \
  -e POSTGRES_USER=admin \
  -e POSTGRES_DB=myapp \
  -v pgdata:/var/lib/postgresql/data \
  -p 5432:5432 \
  postgres:14

# Подключитесь к базе
docker exec -it postgres-db psql -U admin -d myapp

# Создайте таблицу
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

# Выйдите из psql
\q

📖 Пример 5: Приложение на Node.js + MongoDB 
# Создайте сеть
docker network create app-network

# Запустите MongoDB
docker run -d \
  --name mongo \
  --network app-network \
  -e MONGO_INITDB_ROOT_USERNAME=root \
  -e MONGO_INITDB_ROOT_PASSWORD=password \
  mongo:6.0

# Создайте простое приложение (app.js)
cat <<EOF > app.js
const express = require('express');
const app = express();
app.get('/', (req, res) => {
  res.send('Hello from Node.js in Docker!');
});
app.listen(3000, () => {
  console.log('Server running on port 3000');
});
EOF

# Создайте package.json
cat <<EOF > package.json
{
  "name": "docker-app",
  "version": "1.0.0",
  "dependencies": {
    "express": "^4.18.0"
  }
}
EOF

# Создайте Dockerfile
cat <<EOF > Dockerfile
FROM node:16-alpine
WORKDIR /app
COPY package.json .
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]
EOF

# Соберите образ
docker build -t my-node-app .

# Запустите приложение
docker run -d \
  --name node-app \
  --network app-network \
  -p 3000:3000 \
  my-node-app

# Откройте http://localhost:3000
📖 Пример 6: Создание Docker Compose для полного стека 
# Создайте docker-compose.yml
cat <<EOF > docker-compose.yml
version: '3.8'

services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    depends_on:
      - api

  api:
    build: ./api
    environment:
      DATABASE_URL: postgresql://user:pass@db:5432/myapp
    depends_on:
      - db
      - redis

  db:
    image: postgres:14
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: myapp
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:alpine

volumes:
  pgdata:
EOF

# Запустите весь стек
docker compose up -d

# Проверьте статус
docker compose ps

# Посмотрите логи
docker compose logs -f api

# Остановите всё
docker compose down
💡 Полезные шпаргалки:
  • Быстрый деплой: docker run -d -p 80:80 --name app myapp
  • Бэкап БД: docker exec postgres pg_dump -U user mydb > backup.sql
  • Восстановление БД: docker exec -i postgres psql -U user mydb < backup.sql
  • Обновление контейнера: docker pull image && docker stop app && docker rm app && docker run ...
  • Проверка здоровья: docker inspect --format='{{.State.Health.Status}}' app

🎯 Заключение:

Docker - мощный инструмент для контейнеризации приложений. Эта справка охватывает основные команды, но возможности Docker гораздо шире. Рекомендуем изучить официальную документацию на docs.docker.com для углублённого изучения.

Рекомендуемые практики:

  • Регулярно обновляйте образы для получения патчей безопасности
